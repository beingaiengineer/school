# BAE School Platform Design Specifications (DESIGN.md)

This document defines the platform-level design guidelines, folder layout requirements, and custom components for courses written for the BAE School playbook.

## Multi-Topic Architecture
BAE School is a multi-course engineering platform.
All courses are organized inside the `docs/school/` directory. Each course gets its own dedicated subdirectory.

Existing Courses:
- `python/`
- `agentic-frameworks/`
- `app-security/`
- `cicd-pipelines/`
- `fastapi-concepts/`
- `git-workflows/`
- `incident-management/`
- `observability/`
- `oop-patterns/`
- `release-deployment/`
- `requirements-estimation/`
- `sdlc-agile/`
- `sli-slo/`
- `software-testing/`
- `team-structure/`

Each course track must:
1. Define its curriculum map inside `docs/school/<course-name>/design.md`.
2. Follow the global layout tokens and components.
3. Hook into the autogenerated Docusaurus sidebar configurations.

## Design Token Standards
BAE School is styled using an Apple-inspired Minimalist Black & White theme defined in `src/css/custom.css`. Make sure all pages, custom components, and custom blocks follow these tokens:

- **Global Layout (Light Mode)**: Pure white background (`#ffffff`), surface `#f5f5f7`, content/heading `#1d1d1f`.
- **Global Layout (Dark Mode)**: Pure obsidian black background (`#000000`), surface `#1d1d1f`, content/heading `#f5f5f7` or `#ffffff`.
- **Primary Accents**: Vary by course (e.g., Python uses `#1d1d1f` in light mode, `#f5f5f7` in dark mode).
- **Typography Scale**: The body and heading font is `Geist` (or falls back to `Inter`/system-ui) and code blocks use `95%` size.
- **Scrollbars & Polish**: Uses glassmorphism (`backdrop-filter: blur(12px)`) for navbars, custom slim scrollbars.

*(Note: If older documents reference "True Crimson `#9f1239`", that design is deprecated. Follow the Black & White tokens).*

## Custom Layout Components
- **`<LearningFlow>`**: Every technical concept page must embed this custom component to visualize flowcharts and system connections.
  - Exposes strict, type-safe props: `steps?: FlowStep[]`, `diagram?: Diagram`.
  - Keep properties optional where sensible, and document default states.
- **`<Admonition>`**: Use Docusaurus caution/info blocks strategically for senior production notes.

## MDX Conventions
- Run `npm run typecheck` to verify React/TypeScript components.
- Run `npm run build` to confirm Docusaurus can successfully compile all static assets without syntax or formatting errors.
