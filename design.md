# BAE School Platform Design Specifications

This document defines the platform-level design guidelines, folder layout requirements, and validation standards for courses written for the BAE School playbook.

---

## Multi-Topic Architecture
BAE School is a multi-course engineering platform.
All courses are organized inside the `docs/school/` directory. Each course gets its own dedicated subdirectory:
```
docs/
├── school/
│   ├── python/                # Python Backend & System Design
│   ├── agentic-frameworks/    # AI Agent Frameworks & Patterns
│   ├── javascript/            # JavaScript & Engine Internals (Future)
│   ├── devops/                # Cloud Infrastructure & Kubernetes (Future)
│   └── <course-name>/         # New course tracks
```

Each course track must:
1. Define its curriculum map inside `docs/school/<course-name>/design.md`.
2. Follow the global layout tokens and components.
3. Hook into the autogenerated Docusaurus sidebar configurations.

---

## Design Token Standards
BAE School is styled using the custom **True Crimson** design system. Make sure all pages, custom components, and custom blocks follow these tokens:
- **Obsidian Black Base**: Dark mode background must be pitch black (`#0a0a0a`), text `#f6ddde`, and borders `#404040`.
- **True Crimson Highlight**: Primary accents are set to `#9f1239` (rose-800).
- **Secondary Surfaces**: Card and panel backgrounds use flat dark charcoal (`#141414` / `#111111` / `#1c1c1c`).
- **Typography Scale**: The body font is `Geist` (or falls back to `Inter`) and code blocks use `JetBrains Mono`.

---

## Custom Layout Components
- **`<LearningFlow>`**: Every technical concept page must embed this custom component to visualize flowcharts and system connections.
- **`<Admonition>`**: Use Docusaurus caution/info blocks strategically for senior production notes.

---

## Content Verification Workflow
After making additions or edits:
1. **Typecheck**: Run `npm run typecheck` to verify React/TypeScript components.
2. **Build**: Run `npm run build` to confirm Docusaurus can successfully compile all static assets without syntax or formatting issues.
