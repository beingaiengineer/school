# Metaclasses

## 1. Quick Summary
Bro, if an Object is an instance of a Class, then a Class is an instance of a **Metaclass**.
Metaclasses allow you to intercept the creation of *classes themselves*. You use them when you want to enforce rules, automatically add methods, or modify the behavior of a class before it even exists in memory.

## 2. Engineering Story
Bro, imagine you are building a strict API framework. You want to enforce that every single API view class developers create *must* define a `handle_request` method. If they forget, you want the application to crash on startup, not when a user actually hits the endpoint. To enforce class-level rules before an object is even instantiated, you need metaclasses—the classes that create classes.

## 4. Concept Explanation
In Python, everything is an object, including classes. The default metaclass that creates all classes is called `type`.

When Python reads a class definition (`class MyClass:`), it gathers the class name, its parent classes (bases), and all its methods/attributes (dictionary). It then passes these three things to the Metaclass (`type(name, bases, dict)`).

By writing your own Metaclass (inheriting from `type`), you can hook into this `__new__` method and mutate the class dictionary before the class is officially constructed.

## 5. Syntax Table

| Method | Signature | Purpose |
|---|---|---|
| `class Meta(type):` | Inherits from `type`. | This is how you define a Metaclass. |
| `__new__` | `(mcs, name, bases, dct)` | Modifies the class dictionary *before* creation. |
| `__init__` | `(cls, name, bases, dct)` | Initializes the class *after* creation. |
| `metaclass=` | `class A(metaclass=Meta):` | Tells Python to use your custom metaclass instead of `type`. |

## 6. Beginner Example
Let's build a strict supervisor. We want a Metaclass that crashes the program if a developer tries to create a class without a docstring.

```python
class RequireDocstringMeta(type):
    # mcs = the metaclass itself
    # name = "MyAPI"
    # bases = ()
    # dct = {"__module__": "...", "do_work": <function>}
    def __new__(mcs, name, bases, dct):
        # We don't care about magic methods, just normal classes
        if not dct.get('__doc__'):
            raise TypeError(f"Bro! The class '{name}' is missing a docstring!")

        # Pass the (modified) data up to the default type constructor
        return super().__new__(mcs, name, bases, dct)

# This will work perfectly
class GoodClass(metaclass=RequireDocstringMeta):
    """This class is well documented."""
    raise NotImplementedError("This method must be implemented by the subclass")
# This will crash IMMEDIATELY when the file is run, before any objects are made!
# class BadClass(metaclass=RequireDocstringMeta):
#     raise NotImplementedError("This method must be implemented by the subclass")
# TypeError: Bro! The class 'BadClass' is missing a docstring!
```

## 7. Real-World Engineering Example
Bro, Django Models use Metaclasses heavily. When you define `email = models.CharField()`, how does the model know the name of the database table?
The Metaclass loops through the attributes, finds all the fields, registers them in an internal registry, and creates the table schema behind the scenes.

Let's build a simple Metaclass that automatically converts all method names to lowercase, so if a Java dev tries to use `camelCase`, we fix it automatically.

```python
class SnakeCaseEnforcerMeta(type):
    def __new__(mcs, name, bases, dct):
        new_dct = {}
        for attr_name, attr_value in dct.items():
            if not attr_name.startswith('__'):
                # Force lowercase
                clean_name = attr_name.lower()
                new_dct[clean_name] = attr_value
            else:
                new_dct[attr_name] = attr_value

        return super().__new__(mcs, name, bases, new_dct)

class JavaDev(metaclass=SnakeCaseEnforcerMeta):
    def doWorkToday(self):
        return "Working..."

dev = JavaDev()
# dev.doWorkToday()  <-- AttributeError!
print(dev.doworktoday()) # Working... (The metaclass renamed it!)
```

## 8. Internal Working
The flow of Metaclasses is confusing for beginners because of the layers of instantiation.
1. `__call__` on the Metaclass is triggered when you do `MyClass()`. It controls object instantiation!
2. `__new__` on the Metaclass allocates memory for the *Class Object*.
3. `__init__` on the Metaclass initializes the *Class Object*.
Only AFTER all of this does `__new__` and `__init__` on the actual class run to create a user object.

## 9. Performance Table

| Pattern | Speed at Runtime | Speed at Import Time | Note |
|---|---|---|---|
| **Normal Class** | Fastest | Fastest | Standard `type` creation. |
| **Metaclass** | Fastest | Slower | Extra logic runs once when the script is imported. Object creation remains fast. |
| **`__init_subclass__`** | Fastest | Slower | A modern, simpler alternative to metaclasses for basic hooks. |

## 10. Top Interview Questions

| Question | Answer | Explanation |
|---|---|---|
| What is a Metaclass? | A class whose instances are classes. | Just like an object is created by a class, a class is created by a metaclass (default is `type`). |
| When should you use a Metaclass? | Almost never. | As Tim Peters (Python core dev) said: "Metaclasses are deeper magic than 99% of users should ever worry about." Use them only for complex frameworks (like ORMs). |
| What is the difference between `type` and `object`? | `object` is the base of all inheritance. `type` is the base of all creation. | `type` inherits from `object`, but `object` is an instance of `type`. (Brain explosion). |

## 11. Tricky Questions & Edge Cases
**Q: I have Class A with Metaclass X, and Class B with Metaclass Y. I want Class C to inherit from both A and B. What happens?**
Bro, you will get a `TypeError: metaclass conflict`. Python cannot figure out which metaclass should construct Class C. You have to create a new Metaclass Z that manually inherits from both X and Y to resolve the conflict. This is why overusing metaclasses creates nightmares.

**Q: Can I achieve Metaclass behavior without Metaclasses?**
Yes! Python 3.6 introduced `__init_subclass__`. If you just want to register subclasses or check attributes, you can define `__init_subclass__` in the parent class. It is vastly simpler and avoids metaclass conflicts.

## 12. Real-World Usage
- **Singletons:** You can write a Metaclass whose `__call__` method ensures only one instance of the class is ever created.
- **Abstract Base Classes:** Python's `abc.ABCMeta` uses a metaclass to prevent you from instantiating a class if it hasn't implemented the required abstract methods.
- **API generation:** Frameworks like Pydantic or FastAPI reading class type hints and generating JSON schemas on the fly.

## 13. Best Practices
- **Don't use them.** Seriously, if you can solve the problem with decorators, standard inheritance, or `__init_subclass__`, do that instead.
- **Keep it Simple:** If you must use one, document exactly what it is mutating in the `__new__` method, because it is invisible to other developers.
- **Inherit from `type`:** Never forget to have your metaclass inherit from `type`, or it won't be able to construct the class properly.

## 14. Production Notes
Metaclasses run at *import time*. If you put a heavy network call or complex algorithm inside a Metaclass `__new__` method, your entire application will freeze the moment someone tries to `import` your file, long before `main()` even starts.

## 15. Common Mistakes
- **Confusing `__new__` in metaclass vs class:** In a metaclass, `__new__` creates the *class*. In a regular class, `__new__` creates the *object*.
- **Forgetting to return:** In `__new__`, if you forget to `return super().__new__(...)`, your class will silently evaluate to `None`.
