# 2. Real-World Analogy

Think of driving a car. You know how to use the steering wheel, the accelerator, and the brakes. You don't need to know the complex internal combustion engine mechanics or how the transmission switches gears to drive the car.

The car provides you with a simple, abstracted interface (the pedals and steering wheel) while hiding the complex implementation details (the engine and gearbox). Bro, that's exactly what abstraction is in code!

---

# 3. Concept Explanation

Abstraction is an OOP concept that focuses on exposing only the necessary parts of an object while hiding the underlying complexity.

In Python, we achieve abstraction using **Abstract Base Classes (ABCs)**. An abstract class is like a blueprint for other classes. It defines methods that the child classes *must* implement, but it doesn't always provide the implementation itself. You cannot create an object directly from an abstract class.

Bro, we use abstraction to enforce a strict contract. If you say a class is a "PaymentProcessor", you force it to have a `process_payment()` method.

# 5. Beginner Example

Let's look at a simple shape example.

```python
from abc import ABC, abstractmethod

# 1. Create an abstract class
class Shape(ABC):

    @abstractmethod
    def area(self):
        pass # No implementation here!

# 2. Child classes MUST implement the area method
class Square(Shape):
    def __init__(self, side):
        self.side = side

    def area(self):
        return self.side * self.side

class Circle(Shape):
    def __init__(self, radius):
        self.radius = radius

    def area(self):
        return 3.14 * self.radius * self.radius

# 3. Usage
sq = Square(4)
print(sq.area()) # Output: 16

# This would crash! You can't instantiate Shape directly.
# s = Shape() # TypeError: Can't instantiate abstract class Shape
```

# 6. Real-World Engineering Example

In production, abstraction is heavily used for plugins, adapters, or service layers. Here is how we might abstract cloud storage.

```python
from abc import ABC, abstractmethod

class CloudStorage(ABC):
    @abstractmethod
    def upload(self, file_name: str, data: bytes):
        pass

    @abstractmethod
    def download(self, file_name: str) -> bytes:
        pass

class AWSStorage(CloudStorage):
    def upload(self, file_name: str, data: bytes):
        print(f"Uploading {file_name} to AWS S3 bucket...")
        # AWS specific code here

    def download(self, file_name: str) -> bytes:
        print(f"Downloading {file_name} from AWS S3...")
        return b"aws_data"

class GCPStorage(CloudStorage):
    def upload(self, file_name: str, data: bytes):
        print(f"Uploading {file_name} to Google Cloud Storage...")
        # GCP specific code here

    def download(self, file_name: str) -> bytes:
        print(f"Downloading {file_name} from GCP...")
        return b"gcp_data"

def backup_database(storage: CloudStorage, data: bytes):
    # The backup logic doesn't care if it's AWS or GCP!
    # It just relies on the abstracted .upload() method.
    storage.upload("db_backup.sql", data)

# Works perfectly with either!
aws = AWSStorage()
backup_database(aws, b"my db data")
```

# 7. Internal Working

In Python, the `abc` module stands for Abstract Base Classes. It uses a special metaclass called `ABCMeta`.

When Python loads a module, it checks if a class inherits from `ABC` and contains methods decorated with `@abstractmethod`. If you try to instantiate a subclass, Python intercepts the creation process. It checks if the subclass has overridden all abstract methods. If even one is missing, it raises a `TypeError` before the object is ever created in memory.

# 8. Performance Table

| Action | Performance Impact | Note |
|---|---|---|
| Inheriting `ABC` | Near Zero | Very slight overhead during class definition, none at runtime. |
| Method Dispatch | O(1) | Same speed as a normal method call. |
| Instantiating | Fast | Once the class passes the abstract check, it instantiates normally. |

# 9. Top Interview Questions

| Question | Answer |
|---|---|
| Can an abstract class have normal methods? | Yes, an abstract class can have fully implemented concrete methods alongside abstract methods. |
| What happens if a child class forgets to implement an abstract method? | Python will throw a `TypeError` when you try to instantiate the child class. |
| Why use `abc` instead of just raising `NotImplementedError`? | Raising `NotImplementedError` only fails when the method is *called*. `ABC` fails at *instantiation*, which is safer and catches bugs earlier. |

---

# 10. Tricky Questions & Edge Cases

The `NotImplementedError` trap!

```python
class OldSchoolStorage:
    def upload(self):
        raise NotImplementedError("Subclasses must implement this!")

class BadStorage(OldSchoolStorage):
    pass # Forgot to implement upload!

# Tricky: This doesn't crash during instantiation!
bad = BadStorage()

# It only crashes when called! Very dangerous in production.
# bad.upload()
```

Bro, this is why we use `ABC` and `@abstractmethod`. The old way of raising `NotImplementedError` is risky because the bug hides until the code actually runs that specific path.

# 11. Real-World Usage

- **Database Connectors:** Django defines abstract database backends, and PostgreSQL/MySQL adapters implement the concrete details.
- **Payment Gateways:** Systems define a generic `PaymentProvider`, and Stripe/PayPal/Razorpay implement the actual logic.
- **Data Pipelines:** An `ETLJob` abstract class ensures every new data pipeline has `extract()`, `transform()`, and `load()` methods.

---

# 12. Best Practices

| DO | DON'T |
|---|---|
| Use the `abc` module to strictly enforce contracts. | Rely on raising `NotImplementedError` in base classes. |
| Keep abstract interfaces small and focused. | Put 50 unrelated abstract methods in one giant base class. |

# 13. Production Notes

> ⚠ Production Note:
In large enterprise codebases (like clean architecture or hexagonal architecture), abstraction is how you decouple your core business logic from external frameworks. Your core logic should only depend on Abstract Classes. This makes unit testing incredibly easy because you can pass in a "MockStorage" that implements the `CloudStorage` abstract class without needing real AWS credentials.

---

# 14. Common Mistakes

- **Forgetting to inherit from `ABC`:** You might use `@abstractmethod` but forget to inherit `ABC`. If you do this, Python won't enforce anything!
- **Instantiating the base class:** Trying to do `my_obj = MyAbstractClass()` will crash your program.
- **Typo in the method name:** If the parent has `@abstractmethod def do_work()`, and the child writes `def do_worrk()`, the child won't instantiate.
