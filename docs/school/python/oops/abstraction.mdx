## 4. Concept Explanation

Abstraction is a core OOP concept that focuses on hiding the internal complexities of an object and only revealing the necessary interactions. In Python, abstraction is typically implemented using Abstract Base Classes (ABCs) from the `abc` module.

An abstract class serves as a blueprint. It can contain abstract methods (methods that are declared but contain no implementation) and concrete methods (methods with implementation). You cannot instantiate an abstract class directly. Instead, subclasses must inherit from the abstract class and provide implementations for all abstract methods.

This enforces a strict contract, bro. If a subclass doesn't implement the abstract methods, it will also be considered abstract and cannot be instantiated.

## 6. Beginner Example

Let's look at a simple example of shapes.

```python
from abc import ABC, abstractmethod

class Animal(ABC):
    @abstractmethod
    def make_sound(self):
        pass

    def sleep(self):
        print("Sleeping... Zzz")

class Dog(Animal):
    def make_sound(self):
        return "Woof!"

class Cat(Animal):
    def make_sound(self):
        return "Meow!"

# animal = Animal()  # ❌ TypeError: Can't instantiate abstract class Animal

dog = Dog()
print(dog.make_sound())  # Output: Woof!
dog.sleep()              # Output: Sleeping... Zzz
```

Bro, notice how `Animal` forces `Dog` and `Cat` to implement `make_sound()`, but allows them to share the `sleep()` implementation.

## 7. Real-World Engineering Example

In production systems, abstraction is heavily used for defining plugins, drivers, or handlers. For example, a payment processing system might support multiple payment gateways.

```python
from abc import ABC, abstractmethod

class PaymentGateway(ABC):
    @abstractmethod
    def process_payment(self, amount: float) -> bool:
        """Process the payment and return True if successful."""
        pass

    @abstractmethod
    def refund_payment(self, transaction_id: str) -> bool:
        """Refund a payment given a transaction ID."""
        pass

class StripeGateway(PaymentGateway):
    def process_payment(self, amount: float) -> bool:
        print(f"Processing ${amount} via Stripe API...")
        # Complex Stripe API logic here
        return True

    def refund_payment(self, transaction_id: str) -> bool:
        print(f"Refunding {transaction_id} via Stripe...")
        return True

class PayPalGateway(PaymentGateway):
    def process_payment(self, amount: float) -> bool:
        print(f"Processing ${amount} via PayPal API...")
        # Complex PayPal API logic here
        return True

    def refund_payment(self, transaction_id: str) -> bool:
        print(f"Refunding {transaction_id} via PayPal...")
        return True

# Our checkout system doesn't care which gateway is used!
def checkout(gateway: PaymentGateway, amount: float):
    if gateway.process_payment(amount):
        print("Checkout successful!")

checkout(StripeGateway(), 100.0)
```

## 8. Internal Working

Python doesn't have true interfaces like Java. Instead, it uses the `abc` module, which utilizes a metaclass called `ABCMeta`. When a class has `ABCMeta` as its metaclass and defines methods with the `@abstractmethod` decorator, Python registers these methods.

During instantiation (`__new__`), Python checks if the class or any of its parent classes have unimplemented abstract methods. If any are found, it raises a `TypeError` preventing the creation of the object.

| Operation | Complexity | Note |

| Memory Overhead | Minimal | Negligible overhead from storing `__abstractmethods__` set |

## 10. Top Interview Questions

| Question | Answer |
|---|---|
| How do you enforce abstraction in Python? | By using the `abc` module, subclassing `ABC`, and using the `@abstractmethod` decorator. |
| Can an abstract class have concrete methods? | Yes, an abstract class can have fully implemented (concrete) methods that subclasses can inherit or override. |
| What happens if a subclass doesn't implement an abstract method? | The subclass is also considered abstract and any attempt to instantiate it will raise a `TypeError`. |
| Can you instantiate an abstract class? | No, attempting to instantiate a class with at least one unimplemented abstract method raises a `TypeError`. |
| Difference between Encapsulation and Abstraction? | Encapsulation hides data (using private/protected attributes). Abstraction hides complexity by defining strict contracts (interfaces). |

## 11. Tricky Questions & Edge Cases

Bro, look out for this tricky interview question. What happens if you define an `__init__` in an abstract class?

```python
from abc import ABC, abstractmethod

class BaseDatabase(ABC):
    def __init__(self, connection_string):
        self.connection_string = connection_string

    @abstractmethod
    def connect(self):
        pass

class MySQLDatabase(BaseDatabase):
    def connect(self):
        print(f"Connecting to MySQL at {self.connection_string}")

db = MySQLDatabase("mysql://localhost")
db.connect() # This works perfectly!
```

**Trick:** Yes, abstract classes can have `__init__` methods! Subclasses must still call `super().__init__()` if they override it, or they'll inherit it normally. You just can't instantiate the abstract class *itself*.

## 13. Best Practices

| DO | DON'T |
|---|---|
| Use `abc` module to strictly enforce contracts. | Don't rely on raising `NotImplementedError` in regular base classes if you want strict enforcement at instantiation time. |
| Keep abstract base classes focused and small (Interface Segregation Principle). | Don't create massive "god" abstract classes with dozens of abstract methods. |

## 14. Production Notes

> ⚠ Production Note:
Relying solely on `raise NotImplementedError("Subclass must implement")` inside a normal class is an older Python pattern. It only fails when the method is *called*. Using `ABC` and `@abstractmethod` fails fast at *instantiation*, which is much safer for production systems. It prevents a half-baked object from causing a crash deep within your application pipeline.

## 15. Common Mistakes

- **Forgetting to inherit from `ABC`:** Just using `@abstractmethod` without inheriting from `ABC` does nothing. The class will still be instantiable!
- **Not implementing all abstract methods:** Leaving out even one abstract method makes the subclass abstract.
- **Using `NotImplemented` instead of `NotImplementedError`:** Returning `NotImplemented` is for rich comparisons (`__eq__`), while `NotImplementedError` is an exception you raise.
