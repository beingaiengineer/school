# FastAPI

## 1. Quick Summary

| Area | Details |
|---|---|
| Topic | FastAPI |
| Difficulty | Intermediate |
| Used For | Building high-performance, async Python web APIs |
| Main Strength | Automatic swagger docs, type validation (Pydantic), and high speed |
| Common Mistake | Running synchronous blocking code in async endpoints |
| Performance | Top tier for Python; comparable to NodeJS and Go in web frameworks |

Bro, FastAPI is the modern rockstar of Python web frameworks. It took the best parts of Flask (simplicity), combined it with Pydantic (data validation), and built it on top of Starlette (async speed).

## 2. Engineering Story

Migrating a slow, synchronous API to FastAPI unlocked high-performance async capabilities and gave us automatic swagger docs.

## 4. Concept Explanation

FastAPI is designed to build APIs quickly with fewer bugs.

It heavily relies on Python type hints. By simply declaring the types of your variables (e.g., `item_id: int`), FastAPI automatically:
1. Validates the incoming request data.
2. Converts the data to the correct Python type.
3. Generates interactive API documentation (Swagger UI).

It also natively supports async/await, allowing it to handle thousands of concurrent requests if they involve waiting for I/O (like a database or external API).

## 6. Beginner Example

```python
from fastapi import FastAPI
from pydantic import BaseModel

app = FastAPI()

# Pydantic model for JSON validation
class Item(BaseModel):
    name: str
    price: float

@app.get("/")
def read_root():
    return {"message": "Hello World"}

@app.post("/items/")
def create_item(item: Item):
    return {"message": f"Created {item.name} for ${item.price}"}

# Run with: uvicorn main:app --reload
```

Bro, try sending a POST request to `/items/` with `{"name": "Laptop", "price": "expensive"}`. FastAPI will automatically return a `422 Unprocessable Entity` because "expensive" is a string, not a float!

## 7. Real-World Engineering Example

In production, you usually use asynchronous endpoints connected to an async database driver.

```python
from fastapi import FastAPI, HTTPException, Depends
from pydantic import BaseModel
import asyncio

app = FastAPI()

class UserCreate(BaseModel):
    username: str
    email: str

# Mock async database call
async def db_save_user(user: UserCreate):
    await asyncio.sleep(0.5) # Simulating DB latency
    if user.username == "admin":
        raise ValueError("Username reserved")
    return {"id": 101, **user.model_dump()}

@app.post("/users/", status_code=201)
async def register_user(user: UserCreate):
    try:
        new_user = await db_save_user(user)
        return new_user
    except ValueError as e:
        # Proper error handling
        raise HTTPException(status_code=400, detail=str(e))
```

## 9. Performance Table

| Area | Performance | Why |
|---|---|---|
| I/O Bound Tasks (DB, APIs) | Excellent | Async event loop allows handling thousands of connections without blocking. |
| CPU Bound Tasks (Math, Image) | Poor (if async) | Event loop blocks; use `Depends` or background tasks to push to a thread pool. |
| Validation Speed | Very High | Pydantic V2 is written in Rust, making it incredibly fast. |

## 10. Top Interview Questions

| Question | Answer |
|---|---|
| What makes FastAPI fast? | It uses ASGI (Starlette) for async I/O and Pydantic (Rust-based) for validation. |
| How does FastAPI do validation? | By leveraging Python type hints and the Pydantic library. |
| Should I use `def` or `async def` in FastAPI? | If you use blocking libraries (like `requests` or `psycopg2`), use `def`. If you use async libraries, use `async def`. |
| What is Uvicorn? | It is an ASGI web server implementation used to run FastAPI applications. |
| How do you handle Dependency Injection? | Using the `Depends()` function in endpoint parameters (great for DB sessions or Auth). |

## 11. Tricky Questions & Edge Cases

### Blocking the Event Loop

Bro, this is the #1 mistake in FastAPI:

```python
# VERY BAD
@app.get("/slow")
async def slow_endpoint():
    time.sleep(5) # Blocks the ENTIRE async event loop
    return {"message": "done"}
```

If you do this, NO OTHER USER can access your API for 5 seconds.
If you must use a blocking function, declare it with regular `def` (FastAPI will automatically run it in an external thread pool), OR use `asyncio.sleep(5)`.

### Missing Type Hints

If you forget type hints, FastAPI cannot validate or generate docs.

```python
# Bad: Missing type hint on user_id
@app.get("/users/{user_id}")
def get_user(user_id):
    raise NotImplementedError("This method must be implemented by the subclass")
```

FastAPI will treat `user_id` as a string, and it won't be documented properly. Always use `user_id: int`.

## 13. Best Practices

| DO | DON'T |
|---|---|
| Use `pydantic` models for all incoming/outgoing data | Read raw `request.json()` manually |
| Use `Depends()` for database sessions and auth | Create global variables for DB connections |
| Use ASGI servers like Uvicorn or Daphne | Try to run FastAPI on Waitress or pure Gunicorn (WSGI) |
| Return `HTTPException` for errors | Return `{ "error": "msg" }` with a 200 OK status |

## 14. Production Notes

> ⚠ Production Note:
> Never run just `uvicorn main:app` in production. You should use a process manager to run multiple workers to utilize all CPU cores. E.g.: `fastapi run` or `uvicorn main:app --workers 4`.

## 15. Common Mistakes

| Mistake | Why It Hurts |
|---|---|
| Using `requests` library in `async def` | It blocks the event loop. Use `httpx` instead. |
| Putting business logic in the router | Makes testing hard. Separate your routes from your core logic/services. |
| Returning DB models directly | Can accidentally expose hashed passwords or internal IDs. Always use response models (`response_model=UserPublic`). |
