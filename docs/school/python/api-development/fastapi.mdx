# FastAPI

## 1. Quick Summary

| Area | Details |
|---|---|
| Topic | FastAPI |
| Difficulty | Intermediate |

## 2. Real-World Analogy

Bro, think of FastAPI like a hyper-efficient modern airport terminal.

Older frameworks (like Flask/Django without async) are like standard security lines: if one person is taking a long time checking their bags, the whole line stops.
FastAPI is like an automated, parallel TSA line. While one passenger is waiting for their bag to be scanned, the agent immediately starts processing the next passenger. Nobody waits unnecessarily. Plus, FastAPI automatically checks everyone's passport (Data Validation via Pydantic) before they even get to the gate.

---

## 3. Concept Explanation

FastAPI is a modern Python web framework used to build REST APIs. It has three killer features:
1. **Speed:** It is built on `Starlette` (for async routing) and `Pydantic` (for data validation), making it one of the fastest Python frameworks available.
2. **Async Native:** It natively supports `async`/`await`, meaning it can handle thousands of concurrent requests without eating up memory.
3. **Auto-Docs:** It automatically generates beautiful, interactive API documentation (Swagger UI) directly from your code.

## 5. Beginner Example

Let's build a simple API that returns a greeting.

```python
from fastapi import FastAPI
import uvicorn

# Create the app instance
app = FastAPI()

# Define a GET route
@app.get("/")
async def root():
    return {"message": "Hello Bro, welcome to FastAPI!"}

# Define a route with a path parameter
@app.get("/users/{user_id}")
async def get_user(user_id: int):
    # FastAPI automatically converts user_id to an int!
    return {"user_id": user_id, "status": "active"}

# Run with: uvicorn filename:app --reload
if __name__ == "__main__":
    uvicorn.run(app, host="0.0.0.0", port=8000)
```

## 6. Real-World Engineering Example

In production, you use Pydantic models to strictly validate incoming JSON payloads (Request Bodies).

```python
from fastapi import FastAPI, HTTPException
from pydantic import BaseModel, EmailStr

app = FastAPI()

# Define the expected JSON structure
class UserCreate(BaseModel):
    username: str
    email: EmailStr  # Automatically validates email format!
    age: int | None = None

# In-memory DB for example
fake_db = []

@app.post("/users/", status_code=201)
async def create_user(user: UserCreate):
    # If the user sends bad data, FastAPI returns a 422 error automatically.
    # We only get here if validation passes.

    if user.username == "admin":
        raise HTTPException(status_code=400, detail="Username reserved, bro.")

    # Save to "DB"
    fake_db.append(user.model_dump())
    return {"message": "User created successfully", "user": user}
```

FastAPI is built on the ASGI (Asynchronous Server Gateway Interface) standard. You run it using an ASGI server like **Uvicorn**.
When a request hits Uvicorn, it passes to FastAPI. FastAPI uses Python type hints (`user_id: int`) to instantly validate the data using Pydantic (written in fast Rust). If validation fails, it immediately returns a 422 HTTP error. If it passes, it converts the raw JSON into a Python object and hands it to your async function.

## 9. Top Interview Questions

| Question | Answer |
|---|---|
| FastAPI vs Flask? | FastAPI is async-first, uses Pydantic for validation, and auto-generates Swagger docs. Flask is older, synchronous by default, and requires extensions for docs and validation. |
| What happens if you run blocking code in an `async def` route? | It blocks the *entire* Asyncio Event Loop. No other requests can be processed until that code finishes. It is a critical performance bug. |
| How do you handle dependencies (like DB connections) in FastAPI? | Using the `Depends` system. FastAPI automatically injects dependencies (like a DB session) into your route function. |
| What is ASGI? | Asynchronous Server Gateway Interface. It's the modern spiritual successor to WSGI, allowing Python web servers to handle async code. |

## 10. Tricky Questions & Edge Cases

Bro, what is the critical difference between Route A and Route B?

```python
import time
import asyncio
from fastapi import FastAPI

app = FastAPI()

# Route A
@app.get("/sync")
def sync_route():
    time.sleep(3) # Blocking I/O
    return {"status": "done"}

# Route B
@app.get("/async")
async def async_route():
    time.sleep(3) # Blocking I/O
    return {"status": "done"}
```

**Why it's tricky:**
Route B is a **disaster**. Because it's `async def`, FastAPI runs it directly on the main event loop. The `time.sleep(3)` freezes the entire server for 3 seconds.
Route A uses a standard `def`. FastAPI realizes it's synchronous and safely pushes it to a background threadpool. It won't block the server!
**Rule:** If your code is blocking (sync DB drivers, heavy CPU), use `def`. If it is non-blocking (asyncio), use `async def`.

## 12. Best Practices

| DO | DON'T |
|---|---|
| Use Pydantic models for all Request Bodies and Response Models. | Extract data manually from `request.json()`. |
| Use Dependency Injection (`Depends()`) for database sessions and authentication. | Use global variables to manage database state. |
| Use `async def` and `await` with async database drivers (like `asyncpg`). | Use `async def` if your database driver is blocking (like standard `psycopg2`). |

## 13. Production Notes

> ⚠ Production Note:
> Never run FastAPI with just `uvicorn main:app` in production. You must use a process manager like Gunicorn with Uvicorn workers (`gunicorn main:app -w 4 -k uvicorn.workers.UvicornWorker`). This ensures you utilize multiple CPU cores while Uvicorn handles the async concurrency within each core.

## 14. Common Mistakes

- **Returning DB Objects Directly:** Returning a SQLAlchemy ORM object directly instead of dumping it through a Pydantic `response_model`, leaking internal DB states or failing serialization.
- **Missing Type Hints:** FastAPI relies entirely on Type Hints. If you omit them, validation and documentation fail to generate.
- **Mixing Async and Sync:** Attempting to `await` a synchronous library, or forgetting to `await` an asynchronous one.
