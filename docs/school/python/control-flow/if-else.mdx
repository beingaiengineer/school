# If Else

## 1. Quick Summary

| Area | Details |
|---|---|
| Topic | Python `if`, `elif`, `else` |
| Difficulty | Beginner |
| Used For | Running specific code only when certain conditions are true |
| Core Idea | Branching the execution flow based on boolean logic |
| Common Mistake | Forgetting the colon (`:`) or messing up the indentation |
| Performance | O(1) decision making |

Bro, your code needs to make decisions. `if-else` is how you tell Python to do one thing under certain conditions, and something else otherwise.

## 2. Engineering Story

In a robust backend system, such as a payment processor, every request cannot take the exact same execution path. When a transaction request hits your API, you must verify if the user's account is active, if there are sufficient funds, and if the transaction isn't flagged by the anti-fraud system. Without a way to branch our logic dynamically, every request would proceed blindly, leading to catastrophic system failures or massive security breaches. We need a way to route execution based on real-time data states.

## 4. Concept Explanation

Python evaluates the condition next to the `if`.
- If it evaluates to `True`, the indented block underneath runs.
- If it's `False`, Python skips that block.
- `elif` (else if) lets you check another condition if the first one failed.
- `else` is the final fallback if nothing else was true.

Python relies entirely on **indentation** (spaces) to know what code belongs inside the `if` block.

## 5. Syntax Table

| Keyword | Purpose | Example |
|---|---|---|
| `if` | Starts the condition check | `if age >= 18:` |
| `elif` | Secondary checks (optional) | `elif age >= 13:` |
| `else` | Fallback action (optional) | `else:` |
| `:` | Mandatory syntax marker | `if True:` |

## 6. Beginner Example

```python
score = 85

if score >= 90:
    print("Grade: A")
elif score >= 80:
    print("Grade: B")
else:
    print("Grade: C")
```

**Output:**
```text
Grade: B
```

## 7. Real-World Engineering Example

Bro, in a real backend system, you use conditionals to validate requests before doing heavy work.

```python
def process_payment(amount, user_status):
    if amount <= 0:
        return "Error: Invalid amount"

    if user_status == "banned":
        return "Error: User is banned"

    # If we get here, the data is valid
    return "Payment successful"
```
Notice how we use `if` to fail early. We don't wrap the whole success logic in an `else`. This is called the "Guard Clause" pattern.

When Python sees an `if`, it evaluates the expression using the object's `__bool__` method. It doesn't have to be explicitly `True` or `False`.

Python considers these values as `False` (Falsy):
- `None`
- `0`, `0.0`
- `""` (empty string)
- `[]`, `{}`, `set()` (empty collections)

Everything else is considered `True` (Truthy).

## 9. Performance Table

| Operation | Complexity | Note |
|---|---|---|
| `if condition` | O(1) | Assuming the condition itself is fast. |
| Deeply nested `if`s | O(1) | Fast, but terrible for human readability. |

## 10. Top Interview Questions

| Question | Answer |
|---|---|
| Does Python have a `switch` statement? | In older versions, no. In 3.10+, we have `match-case`. |
| Is `if items:` safe? | Yes, checking if a list has elements using its truthiness is Pythonic. |
| Can `else` be used without `if`? | No, `else` must follow an `if` (or `for`/`while`/`try`). |
| What happens if multiple `elif`s are True? | Only the **first** True block executes. The rest are skipped. |

## 11. Tricky Questions & Edge Cases

What happens if you use a list instead of a boolean?

```python
items = []

if items:
    print("List has items!")
else:
    print("List is empty!")
```

**Output:**
```text
List is empty!
```

Bro, you don't need to write `if len(items) > 0:`. Just write `if items:`. It's cleaner, faster, and standard practice.

## 12. Real-World Usage

Where do engineers use this?
- **Web APIs:** Returning 404 if a database query returns `None`.
- **Data Engineering:** Skipping rows in a CSV if a column is empty.
- **UI Logic:** Showing a "Logout" button `if` user is authenticated, `else` showing "Login".

## 13. Best Practices

| DO | DON'T |
|---|---|
| Use "Guard Clauses" to exit functions early | Create massive "Arrow Code" by nesting 5 `if`s |
| Rely on Truthy/Falsy (`if not name:`) | Compare to empty values (`if name == "":`) |
| Compare to booleans directly (`if is_active:`) | Write `if is_active == True:` |

## 14. Production Notes

> ⚠ Production Note:
> Deeply nested `if-else` blocks cause bugs because they are hard to test.

If you see an `if` inside an `if` inside an `if`, refactor your code. Extract the inner logic into a separate function, or use dictionary mappings to handle different cases cleanly.

## 15. Common Mistakes

- Using `=` (assignment) instead of `==` (comparison) in the `if` statement (Python will actually throw a SyntaxError to save you).
- Forgetting the colon `:` at the end of the line.
- Messing up the indentation, causing code to run outside the `if` block.
