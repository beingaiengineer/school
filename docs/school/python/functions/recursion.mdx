# Recursion

## 1. Quick Summary

| Area | Details |
|---|---|
| Topic | Python Recursion |
| Difficulty | Intermediate |
| Used For | Solving problems that can be broken down into smaller identical problems |
| Core Idea | A function that calls itself |
| Common Mistake | Forgetting the "Base Case," causing an infinite loop (Stack Overflow) |
| Performance | Slower and uses more memory than standard `for`/`while` loops |

Bro, recursion is when a function calls itself. It hurts your brain at first, but it is the most elegant way to solve tree-like or deeply nested data problems.

## 2. Real-World Analogy

Think of looking up a word in a physical dictionary.

You want to know what "Recursion" means.
The definition says: "See *Recursive*."
So you look up "Recursive." The definition says: "Characterized by *Recursion*."
You are now stuck in an infinite loop!

To fix this, the dictionary needs a **Base Case**—a word whose definition you already understand, so the searching stops.

## 3. Concept Explanation

Every recursive function MUST have two parts:
1. **The Base Case:** The simple condition where the function stops calling itself and actually returns a value.
2. **The Recursive Case:** The part where the function calls itself, but with a *smaller* or *simpler* piece of data, moving closer to the Base Case.

If you mess up the Base Case, the function calls itself forever until Python panics and crashes with a `RecursionError`.

## 4. Syntax Table

| Concept | Explanation | Example |
|---|---|---|
| Function Definition | Standard `def` | `def count_down(n):` |
| Base Case | The stop condition | `if n <= 0: return` |
| Recursive Call | Calling itself | `count_down(n - 1)` |

## 5. Beginner Example

Let's write a simple countdown.

```python
def countdown(n):
    # 1. Base Case (When do we stop?)
    if n <= 0:
        print("Liftoff!")
        return

    # 2. Do the work for this step
    print(n)

    # 3. Recursive Case (Call ourselves with smaller data)
    countdown(n - 1)

countdown(3)
```

**Output:**
```text
3
2
1
Liftoff!
```

## 6. Real-World Engineering Example

Bro, in a real system, you rarely use recursion for math. You use it to parse nested data, like reading a file system folder that contains folders, which contain more folders.

```python
def print_all_files(folder_structure, indent=""):
    """Recursively prints files in a nested dictionary."""

    for name, content in folder_structure.items():
        if isinstance(content, dict):
            # It's a folder! Call ourselves recursively.
            print(f"{indent}📁 {name}")
            print_all_files(content, indent + "  ")
        else:
            # Base Case: It's just a file.
            print(f"{indent}📄 {name}")

# A nested folder structure
drive = {
    "Photos": {
        "Vacation": {"beach.jpg": "image", "sun.jpg": "image"},
        "profile.png": "image"
    },
    "passwords.txt": "text"
}

print_all_files(drive)
```

## 7. Internal Working

When a function calls itself, Python doesn't just loop back to the top. It creates a brand new "Stack Frame" in memory for the new call.

If `countdown(3)` calls `countdown(2)`, the `3` is paused in memory, waiting for `2` to finish. If `2` calls `1`, both `3` and `2` are paused in memory.
This is why recursion uses O(N) memory. If you nest too deep (usually 1000 calls), Python runs out of stack memory and throws a `RecursionError` to protect your computer from crashing.

## 8. Performance Table

| Operation | Complexity | Note |
|---|---|---|
| Time Complexity | O(N) or worse | Depends on the algorithm (Fibonacci is O(2^N)!). |
| Space/Memory | O(N) | Every recursive call takes up stack memory. |
| Tail Recursion | Not Supported | Python does not optimize tail recursion; it will still blow the stack. |

## 9. Top Interview Questions

| Question | Answer |
|---|---|
| What are the two mandatory parts of a recursive function? | The Base Case and the Recursive Case. |
| What causes a `RecursionError` (Stack Overflow)? | Missing a Base Case, or passing data that never reaches the Base Case. |
| Is recursion faster than a `while` loop? | No. Function calls have memory overhead. Loops are faster and use less memory. |
| Why use recursion if loops are faster? | It makes code for traversing Trees and Graphs (like JSON or Folders) infinitely cleaner to write and read. |

## 10. Tricky Questions & Edge Cases

What is the output of this code? (Watch the print order!)

```python
def mystery(n):
    if n <= 0:
        return
    mystery(n - 1)
    print(n)

mystery(3)
```

**Output:**
```text
1
2
3
```

Bro, notice that it printed `1, 2, 3`, not `3, 2, 1`. Why? Because the `print(n)` happens *after* the recursive call. `mystery(3)` paused and waited for `mystery(2)`, which waited for `mystery(1)`. When the base case hit, they resumed in reverse order! This is called "Unwinding the Stack".

## 11. Real-World Usage

- **Web Scraping:** Scraping a webpage, finding links, and recursively calling the scraper on those new links.
- **UI Rendering:** React and browser DOM trees are rendered recursively (a Div contains a Div contains a Span).
- **JSON Parsing:** Flattening deeply nested API responses.

## 12. Best Practices

| DO | DON'T |
|---|---|
| Put the Base Case at the very top of the function | Bury the Base Case at the bottom |
| Ensure every recursive call moves closer to the Base Case | Pass the exact same arguments recursively (Infinite loop!) |
| Use recursion for Trees/Graphs/Nested dicts | Use recursion for simple counting or math (use a `for` loop) |

## 13. Production Notes

> ⚠ Production Note:
> Never use recursion for user-generated data of unknown depth.

If you write a recursive function to parse a JSON payload, and a malicious user sends you a JSON object nested 2000 levels deep, your server will hit the recursion limit, crash, and drop the connection. For deep, untrusted data, use an iterative approach with a `while` loop and an explicit `list` acting as a stack.

## 14. Common Mistakes

- Forgetting the `return` keyword in the recursive step (e.g., doing `mystery(n-1)` instead of `return mystery(n-1)` when returning a calculated value).
- Swapping the base case and the recursive step logic.
- Using recursion to calculate Fibonacci numbers without "memoization" (caching), causing the server to freeze.
