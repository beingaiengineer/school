# 3. Real-World Analogy

Think of a function like a custom t-shirt printing machine.

The **Parameters** are the empty slots on the machine's control panel labeled "Size", "Color", and "Text". It's the blueprint of what the machine expects.

The **Arguments** are the actual values you punch in when you use the machine, like "Large", "Black", and "Hello World".

Bro, parameters are the variables listed when you *define* the function. Arguments are the real data you pass in when you *call* the function.

---

# 4. Concept Explanation

Functions wouldn't be very useful if they did exactly the exact same thing every time. Parameters allow functions to accept inputs, making them flexible and dynamic.

- **Parameter:** The variable name inside the parentheses in the `def` statement.
- **Argument:** The actual value you pass into the function when calling it.

Python supports several ways to pass arguments:
1. **Positional Arguments:** Passed in the exact order they are defined.
2. **Keyword Arguments:** Passed by explicitly naming the parameter, meaning order doesn't matter.
3. **Default Parameters:** Parameters that have a fallback value if you don't provide an argument.
4. **`*args` and `**kwargs`:** Allow passing an arbitrary number of arguments (we'll cover this deeply in advanced topics, but here's a taste).

# 6. Beginner Example

Let's see positional, keyword, and default arguments in action.

```python
# 'name' is a required parameter
# 'greeting' has a default parameter
def say_hello(name, greeting="Hello"):
    print(f"{greeting}, {name}!")

# 1. Positional argument (order matters)
say_hello("Alice")
# Output: Hello, Alice! (Uses default greeting)

# 2. Positional for both
say_hello("Bob", "Hi")
# Output: Hi, Bob!

# 3. Keyword arguments (order doesn't matter)
say_hello(greeting="Yo", name="Charlie")
# Output: Yo, Charlie!
```

# 7. Real-World Engineering Example

Imagine you are building a function to connect to a database in a backend system.

```python
def connect_to_db(host, port=5432, user="admin", use_ssl=True):
    print(f"Connecting to {user}@{host}:{port} (SSL: {use_ssl})")

# Most of the time, we just need the host, defaults handle the rest
connect_to_db("db.production.internal")

# But for local testing, we override specific parameters using keyword args
connect_to_db("localhost", port=5433, use_ssl=False)
```

Bro, notice how default parameters keep the function call clean for the standard use case, but flexible enough for custom cases!

# 8. Internal Working

When you pass arguments to a Python function, Python handles it via "Pass-by-Object-Reference" (sometimes called pass-by-assignment).

Python doesn't copy the value of the argument. It passes a *reference* to the memory address of the object.
- If you pass an **immutable** object (like an integer or string) and change it inside the function, Python creates a new local object. The original stays the same.
- If you pass a **mutable** object (like a list or dictionary) and mutate it inside the function (e.g., `my_list.append()`), it changes the original object outside the function too!

---

# 9. Performance Table

| Action | Complexity | Note |
|---|---|---|
| Passing Int/String | O(1) | Just passing a reference |
| Passing large List | O(1) | Still just passing a reference, Python DOES NOT copy the list |
| Using `*args`/`**kwargs` | O(N) | Python has to pack the arguments into a tuple/dict |

# 10. Top Interview Questions

| Question | Answer |
|---|---|
| Parameter vs Argument? | Parameter is in the definition (`def func(x):`). Argument is the real value passed (`func(5)`). |
| What are keyword arguments? | Passing arguments by explicitly naming the parameter (`func(a=1, b=2)`). |
| Can positional arguments follow keyword arguments? | No. Positional arguments must always come *before* keyword arguments in a function call. |
| What does `*args` do? | Allows a function to accept any number of positional arguments, packing them into a tuple. |

# 11. Tricky Questions & Edge Cases

Bro, this is the most famous Python interview trick question. **The Mutable Default Argument trap.**

```python
# DO NOT DO THIS
def add_item(item, box=[]):
    box.append(item)
    return box

print(add_item("Apple"))  # Output: ['Apple']
print(add_item("Banana")) # Output: ['Apple', 'Banana'] wait, what?!
```

**Why does this happen?** Default arguments are evaluated *only once* when the function is defined, not every time it is called. Since a list is mutable, the same list persists across multiple function calls!

**The Fix:**
```python
def add_item(item, box=None):
    if box is None:
        box = []  # Creates a fresh list every time
    box.append(item)
    return box
```

# 12. Real-World Usage

- **API Requests:** The `requests.get()` library heavily uses keyword arguments for things like `headers`, `timeout`, and `auth` so you only pass what you need.
- **Data Engineering:** In Pandas, functions like `df.drop(columns=['A'], inplace=True)` use keyword arguments (`inplace=True`) to alter function behavior.
- **CLI Tools:** Command line arguments are parsed and passed as arguments to a main `run(host, port, debug)` function.

---

# 13. Best Practices

| DO | DON'T |
|---|---|
| Use `None` for default arguments if you need a mutable type (list, dict). | Never use `[]` or `{}` as default arguments! |
| Put required parameters first, followed by parameters with default values. | Put default parameters before required ones (Python will throw a SyntaxError). |
| Use keyword arguments for clarity if a function takes many boolean flags. | Call `setup(True, False, True)` (nobody knows what those mean). Call `setup(ssl=True, debug=False, log=True)` instead. |

---

# 14. Production Notes

> ⚠ **Production Note:**
> When designing functions for a large codebase or a library, strongly prefer keyword arguments over long lists of positional arguments. If your function takes 5 positional arguments, a developer calling it *will* mess up the order eventually. You can even force keyword-only arguments in Python by putting an asterisk `*` in the parameter list: `def transfer_funds(amount, *, from_account, to_account):`. This prevents devastating bugs where accounts are swapped!

---

# 15. Common Mistakes

- **SyntaxError:** `non-default argument follows default argument`.
  - Bad: `def greet(msg="Hi", name):`
  - Good: `def greet(name, msg="Hi"):`
- **SyntaxError:** `positional argument follows keyword argument`.
  - Bad: `calculate(rate=0.1, 100)`
  - Good: `calculate(100, rate=0.1)`
