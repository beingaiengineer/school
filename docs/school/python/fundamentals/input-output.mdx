# Input Output

## 1. Quick Summary

| Area | Details |
|---|---|
| Topic | Input and Output |
| Difficulty | Beginner |
| Used For | Reading user input and showing results |
| Core Functions | `input()` and `print()` |
| Common Mistake | Forgetting `input()` always returns a string |
| Production Angle | Use logging, APIs, or files instead of raw `print()` |

Bro, input and output are how your program talks to the outside world.

## 2. Real-World Analogy

Think of a railway ticket counter.

The passenger gives input: destination and date. The counter gives output: ticket and fare.

## 3. Concept Explanation

`input()` reads text from the user.

`print()` writes text to the terminal.

Use them for learning, scripts, and small CLI tools. In production services, use structured logs, API responses, files, queues, or databases.

## 4. Syntax Table

| Task | Example |
|---|---|
| Print text | `print("Hello")` |
| Print variable | `print(name)` |
| Read input | `name = input("Name: ")` |
| Convert input | `age = int(input("Age: "))` |
| f-string output | `print(f"Hi {name}")` |
| Separator | `print("A", "B", sep="-")` |

## 5. Beginner Example

```python
name = input("Enter name: ")
print(f"Hello {name}")
```

If the user types `Asha`, output is:

```text
Hello Asha
```

## 6. Real-World Engineering Example

```python
order_id = input("Order ID: ")

print(f"Checking status for {order_id}")
```

This is fine for a CLI prototype. In an API, `order_id` would come from a request body or route parameter.

## 7. Internal Working

`input()` pauses the program until text is entered. It returns a string without the final newline.

`print()` converts values to strings and writes them to standard output.

## 8. Performance Table

| Operation | Performance Note |
|---|---|
| `input()` | Slow because it waits for a human |
| Single `print()` | Fine |
| Many prints in loop | Can slow programs |
| Logging | Better for services |
| File output | Depends on disk and buffering |

## 9. Top Interview Questions

| Question | Answer |
|---|---|
| What does `input()` return? | Always a string. |
| How do you read an integer? | `int(input(...))` with validation. |
| What does `print()` return? | `None`. |
| Should production APIs use `print()`? | Prefer structured logging. |
| What is stdout? | Standard output stream, usually terminal. |

## 10. Tricky Questions & Edge Cases

```python
age = input("Age: ")
print(age + 1)
```

This raises `TypeError` because `age` is a string.

Fix:

```python
age = int(input("Age: "))
print(age + 1)
```

Bro, convert and validate input before using it.

## 11. Real-World Usage

| Context | Input | Output |
|---|---|---|
| CLI tool | terminal arguments | terminal text |
| API | JSON request | JSON response |
| Data job | CSV file | report file |
| Monitoring | metrics stream | dashboard or alert |

## 12. Best Practices

| DO | DON'T |
|---|---|
| Validate user input | Trust raw input |
| Use f-strings for readable output | Build messy string chains |
| Use logging in services | Leave random debug prints |
| Keep CLI prompts clear | Ask vague questions |

## 13. Production Notes

> Production Note:
> `print()` is not observability.

Production logs should include timestamp, severity, request ID, and useful context.

## 14. Common Mistakes

- Forgetting `input()` returns string.
- Printing secrets.
- Leaving debug prints in services.
- Not handling invalid input.
