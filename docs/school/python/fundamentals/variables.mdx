# Variables

## 1. Quick Summary

| Area | Details |
|---|---|
| Topic | Python Variables |
| Difficulty | Beginner |
| Used For | Storing data in memory and giving it a human-readable name |
| Core Mechanism | Variables in Python are *labels* (references) attached to objects |
| Common Mistake | Thinking a variable is a physical box that holds data |
| Performance | Variable assignment is extremely fast (O(1)), it merely updates a reference |

Bro, a variable is the most fundamental building block in programming. It is how you tell the computer, "Hey, remember this piece of data, and let me call it by this specific name later."

## 2. Engineering Story

Imagine you are building the backend for a massive e-commerce site. A user adds an iPhone to their cart. The price is $999. In your code, you need to calculate the tax, the shipping fee, and the final total. If you hardcode the number `999` in twenty different places, your code works... until the price drops to $899. Now you have to manually find and replace twenty hardcoded numbers, and if you miss one, the customer gets charged wrong, and your CTO is yelling at you.

Instead, you create a variable: `item_price = 999`. Now, whenever you need the price, you just use the name `item_price`. When the price drops, you update it in exactly one place. Variables give you the power of dynamic, maintainable logic.

Think of variables in Python like sticky name tags in an Indian post office.

| Post Office Analogy | Python Equivalent |
|---|---|
| The physical box containing a gift | The actual Data Object in memory (e.g., the number `999`) |
| The sticky name tag placed on the box | The Variable Name (e.g., `price`) |
| Writing a new tag for the same box | Two variables pointing to the same object (`a = b = 999`) |
| Moving the tag to a different box | Reassigning the variable (`price = 899`) |

In many languages (like C++), a variable is the actual physical box. In Python, the variable is just the sticky tag. The data lives independently in memory.

## 4. Concept Explanation

To create a variable in Python, you just invent a name, use the equals sign (`=`), and provide a value. You do not need to declare what type of data it holds (like `int x = 5` in Java). Python figures it out automatically based on the value you assign. This is called **dynamic typing**.

Because variables are just labels, you can assign a label to an integer, and later on the next line, peel that label off and stick it onto a string. Python won't complain.

Naming rules are strict but simple:
- Must start with a letter or an underscore.
- Cannot start with a number.
- Can only contain alphanumeric characters and underscores (A-z, 0-9, and _).
- Are case-sensitive (`age`, `Age`, and `AGE` are three different variables).

## 5. Syntax Table

| Action | Python Syntax | Example |
|---|---|---|
| Assignment | `name = value` | `score = 100` |
| Multiple Assignment | `a, b = val1, val2` | `x, y = 10, 20` |
| Chained Assignment | `a = b = value` | `p1 = p2 = 0` |
| Type Hinting (Optional) | `name: type = value` | `is_active: bool = True` |

## 6. Beginner Example

Here is how you use variables to make code readable.

```python
# Bad way (hardcoded, confusing)
print(80000 * 0.20)

# Senior way (using descriptive variables)
base_salary = 80000
tax_rate = 0.20

tax_amount = base_salary * tax_rate
print(f"You owe ${tax_amount} in taxes.")
```

## 7. Real-World Engineering Example

In production, variables often hold configuration values that dictate how an entire system behaves.

```python
# config.py
# These variables control the environment settings
ENVIRONMENT = "production"
MAX_RETRIES = 3
DB_TIMEOUT_SECONDS = 15.5

def connect_to_database():
    print(f"Connecting to {ENVIRONMENT} database...")
    print(f"Will retry {MAX_RETRIES} times if it fails.")
    print(f"Timeout is set to {DB_TIMEOUT_SECONDS}s.")

connect_to_database()
```

## 8. Internal Working

Bro, when you type `x = 10`, what actually happens in RAM? Python creates an integer object `10` somewhere in memory. Then, it creates a string name `"x"` in the current namespace dictionary. Finally, it stores the memory address (reference) of the object `10` next to the name `"x"`.

## 9. Performance Table

| Operation | Complexity | Note |
|---|---|---|
| Variable Assignment (`x = 5`) | O(1) | Extremely fast dictionary lookup and reference binding. |
| Variable Reassignment | O(1) | Just moves the pointer to a new object. |
| Checking memory address (`id(x)`) | O(1) | Instantly returns the CPython memory address. |

## 10. Top Interview Questions

| Question | Answer |
|---|---|
| Are variables in Python statically or dynamically typed? | Dynamically typed. You do not declare the type, and a variable can change types during execution. |
| Is Python pass-by-value or pass-by-reference? | It is pass-by-object-reference. Variables are references to objects. Mutable objects (like lists) can be changed inside functions, while immutable objects (like integers) cannot. |
| What does `a = b` do? | It makes the label `a` point to the exact same object in memory that the label `b` is currently pointing to. It does not create a copy of the object. |
| How do you declare a constant in Python? | Python doesn't have true constants enforced by the compiler. By convention, you use ALL_CAPS (e.g., `MAX_CONNECTIONS = 100`) to signal to other engineers not to change it. |
| What is variable scope? | Scope dictates where a variable can be accessed. A variable created inside a function (local scope) cannot be accessed outside of it. |

## 11. Tricky Questions & Edge Cases

Bro, look at this classic interview trap:

```python
a = [1, 2, 3]
b = a
b.append(4)
print(a)
```
What does `print(a)` output? It outputs `[1, 2, 3, 4]`.

Because variables are just labels, `a` and `b` are two sticky tags on the *exact same box*. When you modify the box through `b`, you are modifying the box that `a` is looking at.

## 12. Real-World Usage

In modern Python codebases (like FastAPI backends), plain dynamic variables are often considered too loose. Senior engineers use **Type Hints** (`user_id: int = 5`) heavily. Python ignores the hints at runtime, but static analysis tools (like `mypy`) use them to prevent you from accidentally passing a string to a function that expects a number, catching bugs before they hit production.

## 13. Best Practices

| DO | DON'T |
|---|---|
| DO use descriptive, readable names (e.g., `user_age`, `total_price`). | DON'T use single letters (`a`, `b`, `x`) unless it's a throwaway loop counter. |
| DO use `snake_case` for variable names as per PEP 8 standards. | DON'T use `camelCase` (that is for Java/JavaScript) or `PascalCase` (that is for Classes). |
| DO use ALL_CAPS for configuration constants. | DON'T use Python reserved keywords (like `list`, `str`, `max`) as variable names. You will overwrite built-in functions. |

## 14. Production Notes

> ⚠️ **Memory Leaks via Global Variables**
> Global variables (variables declared outside of any function) stay in memory for the entire life of the program. If you append data to a global list inside a long-running web server, that list will grow infinitely until the server crashes with an Out Of Memory (OOM) error. Keep variables scoped tightly inside functions whenever possible.

## 15. Common Mistakes

| Mistake | Bad Code | Good Code |
|---|---|---|
| Overwriting Built-ins | `list = [1, 2, 3]` (Now you can never use the `list()` function again) | `my_list = [1, 2, 3]` |
| Meaningless Names | `d = 86400` | `seconds_in_a_day = 86400` |
| Casing violations | `UserAge = 25` | `user_age = 25` |
