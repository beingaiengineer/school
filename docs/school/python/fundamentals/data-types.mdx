# Data Types

## 1. Quick Summary

| Area | Details |
|---|---|
| Topic | Python Data Types |
| Difficulty | Beginner |
| Used For | Representing numbers, text, truth, collections, and missing values |
| Core Idea | Every value is an object with a type |
| Common Mistake | Mixing strings and numbers accidentally |
| Performance | Depends on type and operation |

Bro, data types tell Python what kind of value you are working with and what operations make sense.

## 2. Real-World Analogy

Think of a grocery store bill.

Item name is text, quantity is an integer, price is a decimal-like number, and paid status is true or false.

## 3. Concept Explanation

Python has built-in types for common data:

| Type | Meaning |
|---|---|
| `int` | Whole numbers |
| `float` | Decimal approximation |
| `str` | Text |
| `bool` | `True` or `False` |
| `list` | Ordered mutable collection |
| `tuple` | Ordered immutable collection |
| `dict` | Key-value mapping |
| `set` | Unique values |
| `NoneType` | Missing value |

Use the right type because it affects correctness, readability, and performance.

## 4. Syntax Table

| Type | Example |
|---|---|
| Integer | `count = 10` |
| Float | `price = 99.5` |
| String | `name = "Asha"` |
| Boolean | `is_paid = False` |
| List | `items = ["tea", "coffee"]` |
| Dictionary | `user = {"id": 1}` |
| Set | `unique_ids = {1, 2, 3}` |
| None | `result = None` |

## 5. Beginner Example

```python
name = "Asha"
age = 21
is_active = True

print(type(name))
print(type(age))
print(type(is_active))
```

Output shows `str`, `int`, and `bool`.

## 6. Real-World Engineering Example

```python
order = {
    "order_id": "ord_101",
    "amount": 499.0,
    "items": ["book", "pen"],
    "is_paid": True,
}

print(order["amount"])
```

APIs often pass mixed data types inside dictionaries.

## 7. Internal Working

Every value in Python is an object. Objects carry:

| Part | Meaning |
|---|---|
| Type | What operations are allowed |
| Value | Actual data |
| Identity | Object location identity |
| Reference count | One memory-management signal in CPython |

Bro, `type(value)` checks the object type at runtime.

## 8. Performance Table

| Type / Operation | Typical Cost | Note |
|---|---|---|
| List index | O(1) | Fast positional access |
| List membership | O(n) | Scans values |
| Set membership | O(1) average | Fast lookup |
| Dict lookup | O(1) average | Key must be hashable |
| String concat in loop | Can be costly | Prefer `join` for many pieces |

## 9. Top Interview Questions

| Question | Answer |
|---|---|
| Is everything an object in Python? | Practically yes, values are objects. |
| Difference between list and tuple? | List is mutable, tuple is immutable. |
| Why use set? | Unique values and fast membership. |
| What is `None`? | A singleton meaning no value. |
| Why can float be tricky? | It stores binary approximations. |

## 10. Tricky Questions & Edge Cases

```python
print(0.1 + 0.2 == 0.3)
```

Output:

```text
False
```

Floats are approximate. For money, bro, use integer cents or `Decimal`.

## 11. Real-World Usage

| System | Data Type Usage |
|---|---|
| Banking | integers or decimals for money |
| Food delivery | dicts for order payloads |
| AI systems | lists for tokens, floats for scores |
| Monitoring | numbers for metrics, strings for labels |

## 12. Best Practices

| DO | DON'T |
|---|---|
| Use `dict` for structured payloads | Use random parallel lists |
| Use `set` for membership checks | Scan a large list repeatedly |
| Use `None` for missing values | Use empty string for every missing field |
| Validate external data | Trust API payload types blindly |

## 13. Production Notes

> Production Note:
> Most production data bugs come from unexpected types at system boundaries.

Validate request bodies, config, CSV rows, and third-party API responses.

## 14. Common Mistakes

- Adding strings and numbers directly.
- Using floats for money.
- Assuming API data always has the right type.
- Confusing empty values with `None`.
