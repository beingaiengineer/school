Bro, data types are how Python knows whether `+` means "add these numbers" or "glue these words together."

## 2. Engineering Story

You are building a login system. A user signs up and enters their age as `"25"` in a web form. The form sends that data to your Python backend. You write code that checks: `if age > 18: allow_login()`.

But the server crashes with a massive `TypeError`. Why? Because the web form sent `"25"` as a **string** (text), and you tried to mathematically compare it to `18`, which is an **integer**. Python panicked because comparing words to math doesn't make sense. Understanding data types is the only way to safely move data from the outside world into your core logic.

## 4. Concept Explanation

In Python, everything is an object, and every object has a type. The built-in data types fall into a few main categories:

- **Numeric Types:** `int` (whole numbers), `float` (decimals), `complex` (math stuff).
- **Text Type:** `str` (strings of characters).
- **Boolean Type:** `bool` (`True` or `False`).
- **Sequence Types:** `list`, `tuple`, `range` (ordered collections).
- **Mapping Type:** `dict` (key-value pairs).
- **Set Types:** `set`, `frozenset` (unordered, unique collections).
- **None Type:** `NoneType` (represents the intentional absence of a value).

Because Python is dynamically typed, you don't explicitly say "this is an int." You just write `x = 5` and Python tags it internally. You can check the type of any variable at runtime using the `type()` function.

## 6. Beginner Example

Here is how you define and check basic data types.

```python
# Defining different types
user_age = 30              # int
account_balance = 1500.50  # float
username = "coder_bro"     # str
is_admin = False           # bool

# Checking types
print(type(user_age))        # Output: <class 'int'>
print(type(account_balance)) # Output: <class 'float'>

# Python prevents mixing incompatible types
# print(username + user_age) # CRASH: TypeError: can only concatenate str to str
```

## 7. Real-World Engineering Example

In backend engineering, you often receive data as raw strings from a network request, and you must validate and structure it into correct types before saving it to a database.

```python
def process_payment_payload(raw_data: dict):
    """
    Simulates receiving a JSON payload and ensuring data types are correct.
    """
    # Network data often comes in as strings
    amount_str = raw_data.get("amount", "0.0")
    user_id = raw_data.get("user_id")

    # We must cast the string to a float for math operations
    amount_float = float(amount_str)

    # Check if the user is a premium member using a boolean
    is_premium = raw_data.get("premium", False)

    if is_premium:
        # Apply a discount
        amount_float = amount_float * 0.90

    print(f"Processing ${amount_float} for User ID {user_id}")

# Simulate incoming API payload
payload = {
    "user_id": 105,
    "amount": "49.99",
    "premium": True
}

process_payment_payload(payload)
```

## 9. Performance Table

| Data Type | Memory Overhead | Speed Characteristic |
|---|---|---|
| `int` | ~28 bytes | Extremely fast for math, but unbounded (can hold massive numbers without overflowing). |
| `float` | ~24 bytes | Slower than ints due to floating-point arithmetic complexity. |
| `str` | High (immutable) | Modifying strings creates entirely new objects in memory (slow). |
| `list` | Medium/High | Fast indexing, slow if you insert items at the beginning `O(n)`. |

## 10. Top Interview Questions

| Question | Answer |
|---|---|
| Is Python strongly or weakly typed? | Python is *strongly* typed but *dynamically* typed. It doesn't coerce types implicitly (e.g., `"5" + 2` throws an error, unlike JavaScript). |
| What is the difference between mutable and immutable types? | Immutable types (`int`, `str`, `tuple`) cannot be changed in memory after creation. Mutable types (`list`, `dict`, `set`) can be updated in place. |
| Why shouldn't you use `float` for currency? | Floats suffer from precision issues (e.g., `0.1 + 0.2` equals `0.30000000000000004`). Use the `decimal` module for money. |
| What is the `None` type? | It is a special singleton object in Python used to represent the absence of a value or a null pointer. |
| How do you check if a variable is a specific type? | You should use `isinstance(var, type)` rather than `type(var) == type` because `isinstance` supports inheritance. |

## 11. Tricky Questions & Edge Cases

Bro, what does this output?
```python
x = 0.1 + 0.2
print(x == 0.3)
```
The answer is `False`. Because of how floating-point numbers are represented in binary hardware, `0.1 + 0.2` actually equals `0.30000000000000004`. Interviewers love to use this to see if you understand hardware-level floating-point limitations. Always use `math.isclose()` for float comparisons.

## 12. Real-World Usage

In modern Python, plain dynamic types are dangerous at scale. Companies like Dropbox and Instagram enforce strict **Type Hinting** across their entire codebase.

Instead of `def process(user):`, they write `def process(user: dict) -> bool:`. The code runs exactly the same, but CI/CD pipelines run tools like `mypy` to statically analyze the code and ensure engineers aren't passing lists into functions expecting dictionaries.

## 13. Best Practices

| DO | DON'T |
|---|---|
| DO use `isinstance()` for type checking. | DON'T use `type(x) == y`. |
| DO use the `decimal` module for financial calculations. | DON'T use `float` for money. |
| DO use Type Hints in all function signatures. | DON'T leave function arguments untyped in large projects. |

## 14. Production Notes

> ⚠️ **Memory Limits of Collections**
> Lists and Dictionaries can hold any data type, including other lists and dictionaries. If you nest these too deeply, or load a 5GB JSON file into a single dictionary, you will cause memory spikes that can trigger OOM kills by your server orchestrator (like Kubernetes). Process massive data in streams.

## 15. Common Mistakes

| Mistake | Bad Code | Good Code |
|---|---|---|
| Implicit string concatenation | `print("Age: " + 25)` (Crashes) | `print(f"Age: {25}")` |
| Comparing floats directly | `if price == 10.0:` | `if math.isclose(price, 10.0):` |
| Checking for None wrongly | `if x == None:` | `if x is None:` (Uses identity, not equality) |
