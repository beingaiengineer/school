# Operators

## 1. Quick Summary

| Area | Details |
|---|---|
| Topic | Python Operators |
| Difficulty | Beginner |
| Used For | Calculations, comparisons, logic, membership, and identity checks |
| Core Idea | Operators perform actions on values |
| Common Mistake | Using `is` when you mean `==` |
| Performance | Depends on operand type |

Bro, operators are the small symbols and keywords that make expressions work.

## 2. Real-World Analogy

Think of a tea shop bill.

`+` adds item prices, `>` checks if the bill is above a discount limit, and `and` checks if the customer is eligible.

## 3. Concept Explanation

Operators let you build expressions.

Use them for:

- math
- comparisons
- boolean decisions
- membership checks
- object identity checks

Avoid clever one-line expressions when they hide business logic.

## 4. Syntax Table

| Category | Operators | Example |
|---|---|---|
| Arithmetic | `+ - * / // % **` | `total = price * qty` |
| Comparison | `== != > < >= <=` | `age >= 18` |
| Logical | `and or not` | `is_paid and in_stock` |
| Membership | `in`, `not in` | `"admin" in roles` |
| Identity | `is`, `is not` | `value is None` |
| Assignment | `= += -= *=` | `count += 1` |

## 5. Beginner Example

```python
price = 100
quantity = 3

total = price * quantity
print(total)
```

Output:

```text
300
```

## 6. Real-World Engineering Example

```python
is_paid = True
stock_count = 5
user_role = "premium"

can_ship = is_paid and stock_count > 0 and user_role in {"premium", "standard"}

print(can_ship)
```

This kind of expression appears in checkout, auth, and workflow systems.

Operators call behavior on objects.

For example, `a + b` may call internal methods like `__add__`. That is why `+` adds numbers but joins strings.

## 8. Performance Table

| Operation | Typical Cost | Note |
|---|---|---|
| Number arithmetic | O(1) usually | Big integers can cost more |
| List membership | O(n) | Scans values |
| Set membership | O(1) average | Hash lookup |
| String concat once | O(n) | Creates new string |
| `is None` | O(1) | Identity check |

## 9. Top Interview Questions

| Question | Answer |
|---|---|
| `==` vs `is`? | `==` checks value, `is` checks identity. |
| What does `//` do? | Floor division. |
| What does `%` do? | Remainder operation. |
| Why use `in` with set? | Fast membership checks. |
| What is short-circuiting? | `and` / `or` stop when result is already known. |

## 10. Tricky Questions & Edge Cases

```python
print("1" + "2")
print(1 + 2)
```

Output:

```text
12
3
```

Same operator, different type behavior.

```python
value = None
print(value == None)
print(value is None)
```

Bro, prefer `is None`.

## 12. Best Practices

| DO | DON'T |
|---|---|
| Use `is None` for None checks | Use `== None` |
| Use sets for many membership checks | Use large lists for lookup-heavy checks |
| Break complex logic into named variables | Ship unreadable boolean chains |
| Use parentheses for clarity | Rely on reader remembering precedence |

## 13. Production Notes

> Production Note:
> Operator mistakes often become business-rule bugs.

For important conditions, name the rule:

```python
can_refund = is_paid and days_since_purchase <= 7
```

## 14. Common Mistakes

- Using `is` for string comparison.
- Forgetting integer floor division with `//`.
- Comparing strings that contain numbers.
- Building huge boolean expressions without names.
