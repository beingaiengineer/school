# Type Conversion

## 1. Quick Summary

| Area | Details |
|---|---|
| Topic | Python Type Conversion (Type Casting) |
| Difficulty | Beginner |
| Used For | Changing a value from one data type to another |
| Core Idea | You explicitly tell Python to convert a type, or Python does it automatically |
| Common Mistake | Trying to convert a non-numeric string like `"apple"` to an `int` |
| Performance | Creating a new object takes a tiny amount of time |

Bro, type conversion is simply taking data in one format and turning it into another so you can actually use it.

## 2. Real-World Analogy

Think of exchanging currency at the airport.

You have 100 US Dollars, but the shop only accepts Indian Rupees. The money is the same value, but you need to convert it into the required format before you can buy your chai.

## 3. Concept Explanation

Python has two types of conversion:
1. **Implicit Type Conversion:** Python does this automatically. For example, adding an `int` and a `float` results in a `float` so no data is lost.
2. **Explicit Type Conversion (Type Casting):** You force Python to change the type using built-in functions like `int()`, `str()`, or `float()`.

You use explicit conversion constantly when dealing with APIs, user inputs, and file reading, because external data often arrives as text.

## 4. Syntax Table

| Conversion | Function | Example | Result |
|---|---|---|---|
| To Integer | `int()` | `int("42")` | `42` |
| To Float | `float()` | `float("3.14")` | `3.14` |
| To String | `str()` | `str(100)` | `"100"` |
| To Boolean | `bool()` | `bool("")` | `False` |
| To List | `list()` | `list("cat")` | `["c", "a", "t"]` |

## 5. Beginner Example

```python
# The user inputs a string
age_string = "25"

# We must explicitly convert it to do math
age_next_year = int(age_string) + 1

print("Next year you will be:", str(age_next_year))
```

## 6. Real-World Engineering Example

Bro, when you read data from environment variables, it's always a string. If you have a feature flag or a configuration port, you must cast it.

```python
import os

# "8080" is retrieved as a string from the environment
port_str = os.getenv("PORT", "8080")

# The web server framework needs an integer
port = int(port_str)

print(f"Starting server on port {port}")
```

## 7. Internal Working

When you call `int("42")`, Python doesn't just change a label. It parses the string character by character, calculates the numeric value, and allocates a brand new integer object in memory.

Implicit conversions, like `10 + 2.5`, happen because Python's `__add__` method for integers knows how to safely upgrade itself to a float before doing the math to prevent data loss.

## 8. Performance Table

| Operation | Complexity | Note |
|---|---|---|
| `int("123")` | O(n) | Where n is number of digits. |
| `list(tuple)` | O(n) | Must iterate and create new list. |
| `bool(obj)` | O(1) | Just checks truthiness via `__bool__` or `__len__`. |

## 9. Top Interview Questions

| Question | Answer |
|---|---|
| What happens if you run `int("3.14")`? | It throws a `ValueError`. You must do `int(float("3.14"))`. |
| Is `bool("False")` True or False? | `True`. Any non-empty string is truthy in Python. |
| Difference between explicit and implicit? | Implicit is done safely by Python; explicit is forced by the programmer. |
| What does `list("hello")` do? | Returns `['h', 'e', 'l', 'l', 'o']`. |

## 10. Tricky Questions & Edge Cases

Look at this boolean conversion trap:

```python
is_active_str = "False"

# Bro, look out!
is_active = bool(is_active_str)

print(is_active)
```

**Output:**
```text
True
```

Why? Because `bool()` checks if the string is empty. `"False"` is a string with 5 characters, so it evaluates to `True`. If you are reading booleans from APIs or env vars, compare them explicitly: `is_active_str.lower() == "true"`.

## 11. Real-World Usage

Where do engineers use type casting heavily?
- **Data Engineering:** Converting CSV strings into integers and dates in Pandas.
- **Web Development:** Casting query parameters (e.g., `?page=2`) from strings to integers.
- **Configuration:** Parsing `.env` files into proper types.

## 12. Best Practices

| DO | DON'T |
|---|---|
| Use explicit comparisons for boolean strings (`val == "true"`) | Use `bool("False")` |
| Catch `ValueError` when casting user input | Assume input is perfectly formatted |
| Let Python implicitly convert `int` to `float` in math | Manually cast numbers before adding them |

## 13. Production Notes

> ⚠ Production Note:
> Never blindly cast external data without a `try/except` block or a validation library like `Pydantic`.

If your backend expects an integer ID but a malicious user sends `?user_id=DROP_TABLE`, `int(user_id)` will crash your app with a `ValueError` resulting in an ugly 500 Server Error, instead of a clean 400 Bad Request.

## 14. Common Mistakes

- Forgetting to cast `input()` results.
- Chaining blind casts like `int(some_api_response)` and crashing the script.
- Assuming `bool("0")` is False (it's True!).
