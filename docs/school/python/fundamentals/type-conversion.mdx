# Type Conversion

## 1. Quick Summary

| Area | Details |
|---|---|
| Topic | Python Type Conversion (Type Casting) |
| Difficulty | Beginner |
| Used For | Transforming data from one type (like a string) into another (like an integer) |
| Core Rule | You cannot magically convert data that doesn't make sense (e.g., casting `"apple"` to an integer) |
| Common Mistake | Assuming a float will safely convert to an int without data loss |
| Performance | Creating new objects during conversion adds slight overhead |

Bro, data rarely arrives in the exact format you need. Type conversion is how you reshape data so Python can actually work with it.

## 2. Engineering Story

You are integrating a third-party payment gateway into your app. The API documentation says it returns the transaction amount. You write your logic: `if transaction_amount > 1000: trigger_fraud_alert()`.

In testing, a $5,000 transaction goes through, but the fraud alert doesn't trigger. You dig into the logs and realize the third-party API sent the amount as a string: `"5000"`. Because you didn't explicitly convert that string into an integer or float, your math logic either failed silently or produced unexpected behavior. In engineering, you must never trust the incoming format. You must forcefully convert (cast) it to what you expect before acting on it.

## 4. Concept Explanation

Type conversion (often called casting) comes in two flavors:

1. **Implicit Conversion**: Python does this automatically. If you add an `int` and a `float`, Python knows that forcing the float into an int would lose the decimal data. So, it automatically promotes the `int` into a `float` to safely complete the math.
2. **Explicit Conversion**: You manually force the change using built-in functions like `int()`, `float()`, `str()`, or `list()`. You do this when Python refuses to guess, like trying to add a string to a number.

When you explicitly cast, Python does not physically mutate the original object in memory. It creates a brand-new object of the requested type and returns it.

## 5. Syntax Table

| Conversion Function | What it does | Example | Result |
|---|---|---|---|
| `int(x)` | Converts `x` to an integer | `int("10")` | `10` |
| `float(x)` | Converts `x` to a decimal | `float(10)` | `10.0` |
| `str(x)` | Converts `x` to a string | `str(3.14)` | `"3.14"` |
| `bool(x)` | Converts `x` to True/False | `bool(0)` | `False` |
| `list(x)` | Converts an iterable to a list | `list("cat")` | `['c', 'a', 't']` |

## 6. Beginner Example

Here is explicit conversion in action to fix a common error.

```python
# The Error Scenario
age_str = "25"
# print(age_str + 5) # This crashes!

# The Fix using Explicit Casting
age_int = int(age_str)
next_year = age_int + 1
print(f"Next year you will be {next_year}")

# Implicit Conversion
base = 10      # int
tax = 1.5      # float
total = base + tax # Python automatically converts 'base' to float
print(type(total)) # Output: <class 'float'>
```

## 7. Real-World Engineering Example

When dealing with environment variables (like a Docker `.env` file), all data is imported as strings. You must cast them before initializing your app.

```python
import os

def initialize_database_pool():
    # os.environ always returns strings!
    pool_size_str = os.environ.get("DB_POOL_SIZE", "10")
    debug_mode_str = os.environ.get("DEBUG_MODE", "False")

    # 1. Cast string to int
    pool_size = int(pool_size_str)

    # 2. Safely cast string to boolean
    # bool("False") actually evaluates to True because the string is not empty!
    # We must explicitly check the string content.
    is_debug = debug_mode_str.lower() == "true"

    print(f"Starting DB with pool size: {pool_size} (Type: {type(pool_size)})")
    if is_debug:
        print("Warning: Running in DEBUG mode.")

# Simulate environment variables
os.environ["DB_POOL_SIZE"] = "25"
os.environ["DEBUG_MODE"] = "True"

initialize_database_pool()
```

## 9. Performance Table

| Casting Operation | Speed | Note |
|---|---|---|
| `int(float)` | Very Fast | Just truncates the decimal bytes. |
| `int("string")` | Slow | Must iterate over the string and perform math to build the integer. |
| `list(tuple)` | O(N) | Must iterate over the entire tuple and copy references to a new list. |

## 10. Top Interview Questions

| Question | Answer |
|---|---|
| What happens if you run `int(3.9)`? | It returns `3`. Converting a float to an int simply truncates (cuts off) the decimal. It does not round up. |
| Why does `bool("False")` return `True`? | Because the `bool()` function returns `True` for *any* string that is not completely empty. The string `"False"` has 5 characters, so it is truthy. |
| What is the difference between implicit and explicit conversion? | Implicit is done automatically by Python to prevent data loss (like `int + float`). Explicit is manually commanded by the engineer using functions like `str()`. |
| Can you convert a list into a string? | Yes, `str([1, 2])` will return the exact string representation `"[1, 2]"`. But usually, you want to use `''.join(list)` to combine elements. |
| What error is thrown when `int("apple")` is run? | A `ValueError`. Python recognizes you want an integer, but the specific value provided cannot be parsed into one. |

## 11. Tricky Questions & Edge Cases

Bro, how do you convert an empty list `[]` to a boolean?
```python
x = []
print(bool(x)) # False
```
In Python, empty collections (lists, dicts, sets), the number `0`, and the `None` object all explicitly cast to `False`. Anything that contains data casts to `True`. This is why we can write clean code like `if my_list:` instead of `if len(my_list) > 0:`.

## 12. Real-World Usage

In data engineering with Pandas, type conversion is called "casting." When you import a massive CSV file, Pandas might accidentally read a column of dates as pure strings. Before you can run time-series analysis, you must explicitly cast that entire column into `datetime` objects.

## 13. Best Practices

| DO | DON'T |
|---|---|
| DO wrap user input casting in `try/except` blocks to catch `ValueError`s. | DON'T assume `int(user_input)` will always succeed. |
| DO check string content (`if val.lower() == 'true'`) when casting environment variables to booleans. | DON'T use `bool("False")`. |
| DO use explicit casting when constructing file paths or logs. | DON'T rely on implicit string concatenation. |

## 14. Production Notes

> ⚠️ **Data Loss in Float Conversion**
> Be extremely careful casting massive integers to floats. While Python integers can be infinitely large, Python floats use standard C double-precision hardware formats. If you cast a massive 50-digit integer to a float, you will lose precision at the tail end.

## 15. Common Mistakes

| Mistake | Bad Code | Good Code |
|---|---|---|
| Misunderstanding float to int | `rounded = int(4.9)` (Yields 4) | `rounded = round(4.9)` (Yields 5) |
| Crashing on bad data | `user_id = int(payload["id"])` | `try: user_id = int(payload["id"])` |
| Boolean string trap | `is_active = bool("False")` | `is_active = payload == "True"` |
