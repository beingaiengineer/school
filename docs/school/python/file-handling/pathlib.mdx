# Pathlib

## 1. Quick Summary

| Area | Details |
|---|---|
| Topic | Pathlib |
| Difficulty | Beginner / Intermediate |
| Used For | Handling file and directory paths safely across Windows, Mac, and Linux |
| Core Idea | Treats paths as objects instead of raw strings |
| Common Mistake | Hardcoding string paths like `"C:\\folder\\file.txt"` which breaks on Linux |
| Performance | Fast enough for almost all file system operations |

Bro, before `pathlib`, everyone used `os.path` and glued strings together to make file paths. It was a nightmare. `pathlib` turns paths into smart Python objects so you don't have to worry about Windows vs. Mac slashes.

## 4. Concept Explanation

The `pathlib` module provides a `Path` class. When you create a `Path` object, you can use the `/` operator to join paths together.

It also gives you built-in methods to:
- Check if a file exists (`.exists()`)
- Create directories (`.mkdir()`)
- Get file extensions (`.suffix`)
- Read text directly (`.read_text()`)

It replaces almost everything you used to do with the `os` and `glob` modules.

## 5. Syntax Table

| Operation | Example |
|---|---|
| Create a Path | `p = Path("my_folder")` |
| Join Paths | `file_path = p / "data.txt"` |
| Check if exists | `file_path.exists()` |
| Check if it's a file | `file_path.is_file()` |
| Get file extension | `file_path.suffix` (Returns `'.txt'`) |

## 6. Beginner Example

Here's how you build a path and check if a file exists.

```python
from pathlib import Path

# Create a path object to a folder
folder = Path("documents")

# Join it with a file name using the / operator
my_file = folder / "notes.txt"

# Check if it's actually there
if my_file.exists():
    print("Bro, the file is here!")
else:
    print(f"File not found at: {my_file.absolute()}")
```

## 7. Real-World Engineering Example

In the real world, you often need to find all `.csv` files in a directory and process them. `pathlib` makes scanning directories incredibly clean.

```python
from pathlib import Path

def process_all_csvs(directory_path):
    # Convert string to Path object
    target_dir = Path(directory_path)

    # .rglob() recursively searches for matching files
    for csv_file in target_dir.rglob("*.csv"):
```

## 8. Internal Working

Under the hood, when you instantiate `Path()`, Python checks your Operating System.
If you are on Windows, it creates a `WindowsPath` object. If you are on Mac/Linux, it creates a `PosixPath` object.

When you use the `/` operator (which internally calls the `__truediv__` magic method), `pathlib` intelligently inserts either a `\` (Windows) or a `/` (Linux) based on the specific path object type.

## 9. Performance Table

| Operation | Speed | Note |
|---|---|---|
| `Path(str)` creation | Fast | Negligible overhead |
| `Path.exists()` | Fast | Calls OS `stat` under the hood |
| `Path.rglob()` | Slower | Scanning deep directory trees takes time |

## 10. Top Interview Questions

| Question | Answer |
|---|---|
| Why use `pathlib` over `os.path.join`? | `pathlib` is object-oriented, safer across different OSs, and provides clean methods like `.suffix` and `.name`. |
| What does the `/` operator do in `pathlib`? | It overrides the division operator to join path components safely based on the OS. |
| How do you get the parent directory of a file? | Use the `.parent` property (e.g., `my_file.parent`). |
| Can `open()` accept a `Path` object? | Yes! Since Python 3.6, built-in `open()` accepts `Path` objects directly. |

## 11. Tricky Questions & Edge Cases

**Edge Case: `resolve()` vs `absolute()`**

If you have a relative path like `../config.json`, how do you get the real path?

```python
from pathlib import Path

p = Path("../config.json")

print(p.absolute())
# Just glues current directory and "../config.json" together

print(p.resolve())
# Actually calculates the real, simplified absolute path, resolving symlinks!
```

Bro, always use `.resolve()` when you need the true path to a file.

## 12. Real-World Usage

- **Django/FastAPI Settings:** Defining the base directory of the project dynamically using `BASE_DIR = Path(__file__).resolve().parent.parent`.
- **Data Engineering:** Scanning nested folders for raw data files before processing them into a database.
- **Log Management:** Automatically creating nested directory structures (e.g., `logs/2023/10/`) using `Path("logs/2023/10").mkdir(parents=True, exist_ok=True)`.

## 13. Best Practices

| DO | DON'T |
|---|---|
| Use `pathlib` for all new Python projects. | Hardcode strings like `"C:\\my_app\\data"`. |
| Use `p.mkdir(parents=True, exist_ok=True)` to safely create folders. | Write complex `if not os.path.exists()` logic to create folders. |
| Use `p.read_text()` for quick one-line file reading. | Open/close files manually if you just need the string content immediately. |

## 14. Production Notes

> ⚠ Production Note:
If you are deploying a Python app inside a Docker container (which is Linux based), but developing on Windows, hardcoded paths will crash your app immediately on deployment. `pathlib` completely eliminates this class of production bugs.

## 15. Common Mistakes

- **Concatenating strings with Paths:** `Path("data") + "/file.txt"` will throw an error. You must use the division operator: `Path("data") / "file.txt"`.
- **Forgetting `exist_ok=True`:** If you call `Path("logs").mkdir()` and the folder already exists, your app will crash with a `FileExistsError`. Always pass `exist_ok=True`.
