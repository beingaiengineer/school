# Writing Files

## 1. Quick Summary

| Area | Details |
|---|---|
| Topic | Writing Files |
| Difficulty | Beginner |
| Used For | Saving reports, logs, and exporting data |
| Main Idea | Connect to a file in write or append mode and push strings to it |
| Common Mistake | Using `'w'` mode and accidentally deleting all existing data |
| Production Angle | File operations can fail if disk is full or permissions are denied |

Bro, writing files is how your Python code saves its hard work so it outlives the script execution.

## 2. Real-World Analogy

Think of writing a file like filling out a notebook.

| Notebook Step | Python Step |
|---|---|
| Buy a new notebook / open an old one | Open file in `'w'` (overwrite) or `'a'` (append) mode |
| Write words on the page | Call `file.write('hello')` |
| Close the notebook so ink dries | Call `close()` or let `with` do it |

## 3. Concept Explanation

To save data to disk, you open a file connection, but this time you specify a mode that allows writing (`'w'` for overwrite, `'a'` for append).

When you write to a file, Python buffers (groups) your strings in memory and flushes them to the hard drive in chunks to improve performance. This is why properly closing the file is critical—if the app crashes before the file is closed, you might lose the buffered data.

## 4. Syntax Table

| Action | Example |
|---|---|
| Overwrite file | `open('data.txt', 'w')` |
| Append to file | `open('data.txt', 'a')` |
| Write string | `file.write("hello\n")` |
| Write list of strings | `file.writelines(["line1\n", "line2\n"])` |
| Best practice write | `with open('data.txt', 'w') as f:` |

## 5. Beginner Example

```python
# The modern, safe way to write a file
with open('report.txt', 'w') as file:
    file.write("Bro, this is the first line.\n")
    file.write("And here is the second.")

print("Report saved!")
```

Notice the `\n`? Python doesn't add newlines automatically when you use `.write()`. You have to do it yourself.

## 6. Real-World Engineering Example

```python
import time

def log_event(event_message):
    # Using 'a' mode ensures we don't delete older logs
    with open('/var/log/my_app.log', 'a', encoding='utf-8') as log_file:
        timestamp = time.strftime("%Y-%m-%d %H:%M:%S")
        log_file.write(f"[{timestamp}] {event_message}\n")

log_event("User logged in")
log_event("Database query successful")
```

Because writing to a physical disk is slow, your OS stores your `.write()` calls in a memory buffer. When the buffer is full, or when you close the file, the OS "flushes" the data to the actual disk.

## 8. Performance Table

| Strategy | Performance Note |
|---|---|
| `file.write()` | Extremely fast for small strings |
| `file.writelines()` | Fast, but avoid building huge lists in memory first |
| Small frequent writes | Slowest due to constant OS interactions |
| Buffered writes | The default and best balance of speed and safety |

## 9. Top Interview Questions

| Question | Answer |
|---|---|
| What happens if you open a file in `'w'` mode? | If the file exists, it is instantly erased (truncated). If it doesn't exist, it's created. |
| How do you add text without deleting what's there? | Use `'a'` (append) mode. |
| Does `write()` add a newline automatically? | No, you must explicitly add `\n`. |
| Why is my written file empty after the script crashes? | Data was in the memory buffer and the file wasn't properly closed/flushed. |
| What does `writelines()` do? | It writes an iterable (like a list) of strings to the file. It does *not* add newlines automatically. |

## 10. Tricky Questions & Edge Cases

```python
data = ["apple", "banana", "mango"]

with open('fruits.txt', 'w') as f:
    f.writelines(data)
```

Bro, if you read this file later, the content is `applebananamango`.
`writelines` is just a loop over `.write()`. It doesn't know about newlines.

## 12. Best Practices

| DO | DON'T |
|---|---|
| Use `with open()` for auto-closing | Rely on garbage collection to close files |
| Explicitly specify `encoding='utf-8'` | Let the OS guess the encoding |
| Use `'a'` mode for logging | Use `'w'` and accidentally wipe your logs |
| Manually add `\n` to your strings | Assume Python formats text files for you |

## 13. Production Notes

> ⚠ Production Note:
> Disk operations can fail. Always anticipate `PermissionError` (you don't have rights to write to the folder) or `OSError` (the disk is full). In robust systems, wrap writes in `try/except` blocks.

## 14. Common Mistakes

| Mistake | Why It Hurts |
|---|---|
| Using `'w'` instead of `'a'` | Accidental data loss by overwriting existing files |
| Writing non-strings | `f.write(100)` crashes. You must cast to string: `f.write(str(100))` |
| Forgetting newlines | Results in one massive, unreadable line of text |
| Hardcoding paths | Breaks if the script is run from a different directory |
