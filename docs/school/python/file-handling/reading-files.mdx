# Reading Files

## 1. Quick Summary

| Area | Details |
|---|---|
| Topic | Reading Files |
| Difficulty | Beginner |
| Used For | Loading configuration, user data, and logs |
| Main Idea | Connect to a file, read its contents into memory, and close it |
| Common Mistake | Forgetting to close the file or loading huge files at once |
| Production Angle | Always use `with open(...)` to handle cleanup automatically |

Bro, reading files is how your Python code connects to the real world of saved data.

## 2. Real-World Analogy

Think of reading a file like borrowing a book from a library.

| Library Step | Python Step |
|---|---|
| Find the book on the shelf | Provide the file path |
| Open the book | Call `open('file.txt', 'r')` |
| Read the pages | Call `read()`, `readline()`, or iterate over it |
| Return the book so others can use it | Call `close()` (or let `with` do it) |

## 3. Concept Explanation

Files are stored on your disk. To use that data in your code, Python needs to open a connection (a file object) to the file, read the bytes, and turn them into strings.

Because file connections are system resources, you must manage them carefully. The best way is using context managers (`with` blocks) so Python handles closing the connection automatically, even if errors happen.

## 4. Syntax Table

| Action | Example |
|---|---|
| Open for reading | `file = open('data.txt', 'r')` |
| Close file | `file.close()` |
| Read everything | `content = file.read()` |
| Read line by line (safe) | `for line in file:` |
| Best practice open | `with open('data.txt', 'r') as f:` |

## 5. Beginner Example

```python
# The modern, safe way to read a file
with open('config.txt', 'r') as file:
    content = file.read()
    print("Bro, here is the config:")
    print(content)
```

By using `with open(...)`, the file closes automatically when the block ends.

## 6. Real-World Engineering Example

```python
def check_error_logs(log_path):
    error_count = 0
    # Reading line-by-line is memory efficient for massive logs
    with open(log_path, 'r') as log_file:
        for line in log_file:
            if "ERROR" in line:
                error_count += 1

    return error_count

total = check_error_logs('/var/log/app.log')
print(f"Bro, we found {total} errors.")
```

When you open a file, your OS creates a file descriptor. If you open too many files without closing them, the OS will reject new requests (the dreaded "Too many open files" error).

## 8. Performance Table

| Strategy | Memory Usage | Use Case |
|---|---|---|
| `file.read()` | High (Loads whole file) | Small config files, scripts |
| `file.readlines()` | High (Creates a list of strings) | Small-to-medium files |
| `for line in file:` | Very Low (One line at a time) | Massive data files, gigabyte logs |
| `file.read(1024)` | Low (Chunking) | Binary files, streams |

## 9. Top Interview Questions

| Question | Answer |
|---|---|
| Why use `with open()`? | It automatically closes the file handle when the block ends, even if an exception occurs. |
| How do you read a 100GB file? | Iterate over it line-by-line using `for line in file:` so you only load one string into RAM at a time. |
| What does `r` stand for? | "Read" mode. It is the default mode. |
| What happens if the file doesn't exist? | `open('file.txt', 'r')` raises a `FileNotFoundError`. |
| What is the difference between `read()` and `readlines()`? | `read()` returns a single massive string. `readlines()` returns a list of strings (lines). |

## 10. Tricky Questions & Edge Cases

```python
with open('data.txt', 'r') as f:
    first_read = f.read()
    second_read = f.read()

print(second_read)
```

Bro, `second_read` will be an empty string `""`.
Why? Reading advances a "cursor" in the file. After the first read, the cursor is at the end. You'd need to use `f.seek(0)` to rewind it.

## 12. Best Practices

| DO | DON'T |
|---|---|
| Use `with open(...)` always | Manually use `f.close()` |
| Specify the encoding `open('x.txt', 'r', encoding='utf-8')` | Rely on system default encodings |
| Catch `FileNotFoundError` | Assume files will always be there |
| Iterate line-by-line for big files | Call `.read()` on unknown file sizes |

## 13. Production Notes

> ⚠ Production Note:
> Always explicitly set `encoding='utf-8'` when reading files in production. Windows defaults to `cp1252` while Linux uses `utf-8`. Your code might work locally on Mac and crash on a Windows server.

## 14. Common Mistakes

| Mistake | Why It Hurts |
|---|---|
| Forgetting encoding | Leads to `UnicodeDecodeError` on different OS platforms |
| Using backslashes incorrectly on Windows | Hardcoding `C:\new\folder` triggers escape character bugs |
| `readlines()` on huge files | Spikes memory and crashes the app |
| Keeping files open too long | Causes file locking issues and OS handle limits |
