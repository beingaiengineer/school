## 2. Real-World Analogy

## 3. Concept Explanation

Tricky Python questions usually come from mutability, references, scope, iterators, truthiness, and runtime behavior.

## 4. Syntax Table

| Pattern | Example |
|---|---|
| Mutable default | `def f(items=[]): ...` |
| Shared reference | `arr = [[]] * 3` |
| Identity check | `x is None` |
| Equality check | `a == b` |

## 5. Beginner Example

```python
arr = [[]] * 3
arr[0].append(1)
print(arr)
```

Output: `[[1], [1], [1]]`

## 6. Real-World Engineering Example

Shared mutable state can leak data between requests, jobs, or tests.

## 7. Internal Working

Names point to objects. Containers store references. Closures read variables from scope when called.

## 13. Production Notes

> Production Note:
> Most tricky Python bugs are simple once you identify shared state, scope, or iterator exhaustion.

## 14. Common Mistakes

- Using mutable defaults.
- Using `is` instead of `==`.
- Reusing exhausted generators.
- Confusing shallow and deep copy.
