## 4. Concept Explanation

Good debugging means reproduce the issue, inspect the traceback, isolate the smallest failing case, and verify the fix.

## 13. Best Practices

| DO | DON'T |
|---|---|
| Read the traceback | Randomly edit files |
| Reproduce first | Guess the fix |
| Add context logs | Log secrets |

## 14. Production Notes

> Production Note:
> A fix is not complete until you can explain how you will detect the bug if it returns.

## 15. Common Mistakes

- Ignoring the exception type.
- Missing import-time failures.
- Using prints instead of logs in services.
- Not writing a regression test.
