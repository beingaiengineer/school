# Software Testing Course Design & Outline Spec

This document defines the curriculum map for the Software Testing course on BAE School.

## Folder Structure

The course must be organized under `docs/school/software-testing/`.

## Topic List

- testing-pyramid.mdx
- unit-testing-with-pytest.mdx
- pytest-fixtures-and-parametrize.mdx
- mocking-and-patching.mdx
- integration-testing.mdx
- api-testing.mdx
- test-driven-development.mdx
- code-coverage.mdx
- load-testing.mdx
- testing-best-practices.mdx

## Content Requirements

Every topic page must follow the global BAE School 16-section template in `CONTENT.md`, the platform voice in `SOUL.md`, and the course-specific rules in `.claude/agents/software-testing-agent.md`.

Each page must also include the course-specific `## Top GitHub Repos` section required by the agent instructions.
