## 3. Concept Explanation

Bro, Agno (which used to be Phidata) is all about giving your LLMs actual memory, knowledge, and tools without the massive boilerplate of older frameworks. It treats agents as independent problem solvers with dedicated storage. It's crucial to understand when to use this versus standard scripts.

## 9. Top Interview Questions

| Question | Answer |
|---|---|
| What makes Agno (formerly Phidata) unique? | Its specific approach to state and tooling. |
| When should I avoid it? | When a simple rule-based script is enough. |
| How do you handle errors? | Try-catch blocks and explicit retry policies. |
| Does it support streaming? | Yes, natively via async generators. |
| How to test it? | Mock the LLM responses. |

## 10. Tricky Questions & Edge Cases

Bro, what happens if the LLM hallucinates a tool call? Agno (formerly Phidata) has specific guardrails, but you must configure them properly or you'll get infinite loops.
