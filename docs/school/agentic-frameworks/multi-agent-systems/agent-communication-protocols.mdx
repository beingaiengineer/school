# Agent Communication Protocols

## 1. Quick Summary

| Area | Details |
|---|---|
| Topic | Agent Communication Protocols |
| Difficulty | Intermediate |
| Used For | Ensuring agents have the right context without overwhelming their prompts. |
| Common Mistake | Passing the entire raw chat history to every single agent in the system. |
| Performance | Evaluated based on token usage and latency. |

## 2. Engineering Story

Bro, if a junior dev asks a senior dev for help, they don't hand over 400 pages of server logs and say 'Read this.' They summarize the issue. When agents communicate, they shouldn't dump their entire context window on each other. We need defined protocols—shared state, message passing, or pub/sub—to communicate efficiently.

## 4. Concept Explanation

Agents must share information. The two main ways are **Shared State** (all agents read/write to a central JSON object, common in LangGraph) and **Message Passing** (agents send specific, targeted messages to one another, common in Actor models).

## 5. Syntax Table

| Protocol | Best For |
|---|---|
| Shared State (Blackboard) | Sequential graphs, single source of truth. |
| Message Passing | Swarms, asynchronous specialized tasks. |

## 7. Real-World Engineering Example

```typescript
// Shared State Example (LangGraph style)
import { StateGraph } from "@langchain/langgraph";

// Define the global blackboard
const AgentState = {
  messages: { value: (x, y) => x.concat(y), default: () => [] },
  scratchpad: { value: (x, y) => ({...x, ...y}), default: () => ({}) }
};

const workflow = new StateGraph({ channels: AgentState });

workflow.addNode("AgentA", async (state) => {
  // Read from shared state, return updates
  return { scratchpad: { agentA_done: true } };
});
```

## 9. Performance Table

| Protocol | Memory Usage | Complexity |
|---|---|---|
| Shared State | High (grows over time) | Low (easy to reason) |
| Message Passing | Low (ephemeral) | High (tracing is hard) |

## 10. Top Interview Questions

| Question | Answer |
|---|---|
| What is the Blackboard pattern? | A shared memory space where all agents can read and write data to collaborate. |
| Why avoid dumping all messages to every agent? | It wastes tokens, increases latency, and confuses specialized agents with irrelevant context. |

## 11. Tricky Questions & Edge Cases

Bro, in a shared state, if two agents write to the same key simultaneously (parallel execution), you get a race condition. Always use reducers to merge state safely.

## 12. Real-World Usage

LangGraph relies heavily on the Redux-like Shared State protocol.

## 13. Best Practices

| Do | Don't |
|---|---|
| Filter messages before passing to a worker | Pass the raw 50-turn chat history to a code formatter |

## 14. Production Notes
