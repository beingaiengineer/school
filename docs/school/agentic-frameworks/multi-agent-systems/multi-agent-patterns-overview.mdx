## 2. Engineering Story

Bro, if you put five senior engineers in a room and tell them to 'build a product' without a manager, a product owner, or a defined workflow, they will just argue about tech stacks for a month. The same happens with LLM agents. Multi-agent systems require architectural patterns—like supervisors, sequential chains, or map-reduce—to actually get work done.

## 4. Concept Explanation

Multi-agent patterns are architectural blueprints for routing tasks, sharing state, and managing execution flow between multiple autonomous agents. As tasks get too complex for a single agent's context window, you split the work. The pattern you choose dictates how these sub-agents communicate.

## 5. Syntax Table

| Pattern | Description |
|---|---|
| Sequential | Agent A -> Agent B -> Agent C. |
| Supervisor | Supervisor routes to workers based on intent. |
| Map-Reduce | Break into chunks, process parallel, summarize. |

## 7. Real-World Engineering Example

```typescript
import { StateGraph } from "@langchain/langgraph";

// Setting up a basic Supervisor workflow graph
const workflow = new StateGraph({
  channels: {
    messages: {
      value: (x, y) => x.concat(y),
      default: () => [],
    },
    nextAgent: {
      value: (x, y) => y,
      default: () => "Researcher",
    }
  }
});

workflow.addNode("Supervisor", supervisorAgent);
workflow.addNode("Researcher", researchAgent);
workflow.addNode("Coder", codingAgent);

// Edges dictate the pattern
workflow.addEdge("Researcher", "Supervisor");
workflow.addEdge("Coder", "Supervisor");
workflow.addConditionalEdges("Supervisor", (state) => state.nextAgent);
```

## 9. Performance Table

| Pattern | Latency | Token Cost |
|---|---|---|
| Sequential | High (Waiting) | Medium |
| Parallel | Low | High |
| Supervisor | Medium | Very High (Supervisor sees all) |

## 10. Top Interview Questions

| Question | Answer |
|---|---|
| Why use multi-agent instead of one big agent? | To bypass context limits and give agents highly specific, focused system prompts. |
| What is the biggest challenge? | Managing state and preventing endless conversation loops. |

## 11. Tricky Questions & Edge Cases

Bro, beware of the 'Polite Loop'. If two conversational agents are instructed to be helpful, they might get stuck in an endless loop of 'Thank you', 'No, thank you'. Always enforce strict stop conditions.
