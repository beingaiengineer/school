## 2. Engineering Story

Bro, you are designing a multi-agent system where a researcher agent needs to pass data to a writer agent, which then hands off the draft to a reviewer agent. In a naive implementation, you might just have them call each other's APIs directly using raw JSON over HTTP. But what happens when the researcher agent crashes mid-thought? Or when the reviewer agent needs to request clarification from the writer? Without a standardized Agent-to-Agent (A2A) protocol, your system quickly devolves into a tangled web of brittle, undocumented point-to-point connections. As an engineer, you need a formal contract that defines how agents discover each other, negotiate capabilities, exchange state, and handle errors gracefully. Standardized A2A protocols solve this by providing a unified communication layer, ensuring your swarm operates reliably at scale.

## 4. Concept Explanation

Bro, the Agent-to-Agent (A2A) protocol is about inter-agent communication. While MCP connects agents to tools, A2A connects agents to OTHER agents. It defines standard message formats so a LangGraph agent can confidently delegate a task to a CrewAI agent. It's crucial to understand when to use this versus standard scripts.

## 10. Top Interview Questions

| Question | Answer |
|---|---|
| Q: What is the main difference between A2A and MCP? | A: A2A focuses on direct communication between peer agents, while MCP connects agents to data sources and tools. |
| Q: How do A2A protocols handle capabilities negotiation? | A: Agents exchange capabilities payloads (often via 'Agent Cards') during the handshake phase to determine supported features. |
| Q: What transport layers are typical for A2A? | A: They commonly use WebSockets for real-time bidirectional communication, or HTTP/REST for asynchronous message passing. |
| Q: How do you handle deadlocks in A2A? | A: By implementing strict timeouts, state tracking, and fallback mechanisms when awaiting a peer's response. |
| Q: Are A2A protocols standardized? | A: While there are emerging standards, many organizations still build custom A2A protocols tailored to their specific orchestrator architectures. |
