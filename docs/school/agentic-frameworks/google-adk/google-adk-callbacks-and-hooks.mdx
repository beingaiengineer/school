## 3. Concept Explanation
Callbacks and hooks in Google ADK allow you to inject custom logic at specific points in the agent lifecycle, such as before a tool is called, after a response is generated, or when an error occurs. This is critical for logging and observability.

## 4. Syntax Table
| Action | Code Example |
|---|---|
| Register Hook | `agent.on("before_tool", my_callback)` |
| Define Callback | `def my_callback(event): print(event)` |

## 5. Beginner Example
```python
def log_tool_usage(event):
    print(f"Tool {event.tool_name} was called.")

agent.on("before_tool_execution", log_tool_usage)
```

## 6. Real-World Engineering Example
```python
def async_audit_log(event):
    # Send to Google Cloud Logging non-blocking
    cloud_logger.log_async(event.data)

agent.register_hook("on_llm_response", async_audit_log)
```

## 9. Top Interview Questions
| Question | Answer |
|---|---|
| Why use hooks instead of modifying the agent code? | Hooks allow separation of concerns, keeping agent logic clean while handling cross-cutting concerns like logging or metrics. |
| What is a common pitfall with callbacks? | Making them synchronous and blocking, which slows down the agent response time. |
| What are the security implications? | You must implement guardrails to prevent unauthorized tool execution or prompt injection. |
| When should you NOT use this? | When a simple direct LLM call or basic script would suffice. |
| How does it scale? | By distributing agent workloads across serverless functions or containerized environments. |

## 10. Tricky Questions & Edge Cases
Watch out for concurrency issues, bro. If multiple agents try to update the same shared state without proper locking mechanisms, you'll run into race conditions that are a nightmare to debug.

## 11. Real-World Usage

## 12. Best Practices
| DO | DON'T |
|---|---|
| Write comprehensive unit tests for individual agent tools | Skip error handling and assume the LLM will always return the perfect format |
| Use structured logging to trace agent decision paths | Hardcode sensitive API keys directly in the agent's configuration |
| Implement clear guardrails and timeout limits | Let an agent run indefinitely in an unbounded loop |

## 13. Production Notes
Warning: Always monitor memory usage and LLM token limits in production. A rogue agent stuck in a loop can quickly rack up massive API bills. Implement hard limits and alerting.

## 14. Common Mistakes
