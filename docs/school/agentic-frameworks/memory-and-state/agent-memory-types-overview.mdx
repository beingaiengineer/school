## 2. Engineering Story

Bro, if you meet a guy at a party and he tells you his name, you remember it for the evening (Short-Term). If you meet him a year later and remember his name, that's Long-Term. If you know that 'fire is hot', that's Semantic. AI agents need the exact same memory hierarchy, otherwise they either forget everything instantly, or they get so overwhelmed with data they go broke paying for tokens.

## 4. Concept Explanation

Memory is how an agent maintains state. **Short-term memory** is simply the array of messages passed to the LLM in the current API call (bound by the context window). **Long-term/Episodic memory** involves persisting those conversations to a database so they can be retrieved later. **Semantic memory** uses embeddings to retrieve factual knowledge (RAG) relevant to the current task.

## 5. Syntax Table

| Memory Type | Implementation |
|---|---|
| Short-Term | `messages` array in API call. |
| Long-Term | Saving `messages` to SQL by `session_id`. |
| Semantic | Vector DB similarity search before prompting. |

## 6. Beginner Example

```typescript
// Short-Term Memory (Context Window)
const history = [
  { role: "user", content: "Hi, I'm Alice." },
  { role: "assistant", content: "Hello Alice!" },
  { role: "user", content: "What is my name?" }
];
// The agent knows the name because it's in the array.
const response = await llm.invoke(history);
```

## 7. Real-World Engineering Example

```typescript
// Complex Memory Orchestration
async function agentWithMemory(userId: string, query: string) {
  // 1. Fetch Semantic Memory (Facts)
  const facts = await vectorDb.search(query, { filter: { userId } });

  // 2. Fetch Long-Term Episodic Memory (Recent chat history)
  const recentChats = await sqlDb.getRecentMessages(userId, 5);

  // 3. Construct Short-Term Memory (Context Window)
  const prompt = [
    { role: "system", content: `Facts: ${facts}
` },
    ...recentChats,
    { role: "user", content: query }
  ];

  const response = await llm.invoke(prompt);

  // 4. Save new interaction to Long-Term memory
  await sqlDb.saveMessages(userId, query, response);
  return response;
}
```

## 9. Performance Table

| Memory | Token Cost | Storage Cost |
|---|---|---|
| Short-Term | High (charged per API call) | None |
| Semantic | Low (only relevant info injected) | Medium (Vector DB hosting) |

## 10. Top Interview Questions

| Question | Answer |
|---|---|
| Why not just put everything in the context window? | Huge context windows (e.g., 1M tokens) are expensive, slow, and models suffer from 'lost in the middle' syndrome where they ignore data in the center of the prompt. |
| What is Semantic Memory? | Retrieving facts based on meaning (embeddings) rather than exact keyword matches. |

## 11. Tricky Questions & Edge Cases

Bro, if your agent summarizes history to save tokens, it will slowly lose granular details over time. It's like a game of telephone. Always keep the raw logs in a database, even if you feed summaries to the LLM.

## 12. Real-World Usage

Memory systems are the core differentiator between a generic chatbot and a personalized AI companion (like ChatGPT's 'Memory' feature).

## 13. Best Practices

| Do | Don't |
|---|---|
| Summarize old conversations | Pass a 100-turn chat history into every request |
