# Task Instructions & Guidelines

**CRITICAL INSTRUCTION FOR ALL AGENTS/DEVELOPERS:**

Before starting any new task, making changes to the codebase, or submitting a commit/PR, you **MUST** always:
1. Read this `task.md` file completely to understand the current task scope and requirements.
2. Read the `MEMORY.md` file completely to understand the historical context, established rules, project standards, and past learnings.
3. Update this `task.md` file and `MEMORY.md` immediately upon completing any topic or task, checking off finished items as `[x]`.
4. Ensure every single page and concept includes a unique, clean React Flow diagram (`LearningFlow`) based on real-world examples.
5. Always create a new git branch for your work, and ensure the Pull Request has a clear, detailed summary of the changes.

Do not proceed with checking or committing code without explicitly acknowledging and reviewing both of these files.
