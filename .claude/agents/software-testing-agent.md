## Tone & Voice

**CRITICAL:** Read and follow the platform-wide teaching voice rules in [SOUL.md](../../SOUL.md) exactly. That is the foundation.

For this course specifically:

### How to Write About Software Testing
Write like a **senior engineer who has been through all of this**. This is NOT academic content. This is survival knowledge. Use unique Indian analogies for Testing: tasting the first sip of chai for sugar (Unit Test) vs verifying a massive wedding catering order (Integration/E2E), doing mock IAS interviews (Mocking), and crash-testing a vehicle (Load Testing).

**Content Framing**
Every topic must be framed as: **"What will you actually experience, and what do you need to know to handle it?"**
